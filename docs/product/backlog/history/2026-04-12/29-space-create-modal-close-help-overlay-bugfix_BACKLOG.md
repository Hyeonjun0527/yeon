## 1차

### 작업내용

- 스페이스 만들기 모달의 닫기(X) 버튼이 동작하지 않는 원인을 추적한다.
- 도움말을 열고 닫은 뒤 스페이스 만들기 버튼이 다시 눌리지 않는 원인을 추적한다.
- 공통 오버레이/모달 상태 정합성을 기준으로 최소 수정안을 적용한다.
- 변경 범위에 대해 진단, 타입, 빌드 검증을 수행한다.

### 논의 필요

- 두 증상이 같은 오버레이 정리 누락에서 발생하는지, 각각 다른 상태 전이 버그인지 확인이 필요하다.
- 공통 Dialog primitive 레벨 수정이 안전한지, 해당 기능 컴포넌트 국소 수정이 더 안전한지 판단이 필요하다.

### 선택지

1. 공통 모달/다이얼로그 primitive 레이어 수정
   - 장점: 같은 유형의 상태 정리 문제를 넓게 차단할 수 있다.
   - 단점: 영향 범위가 커져 다른 모달 회귀 가능성이 있다.
2. 스페이스 생성/도움말 플로우의 open state 및 cleanup만 국소 수정
   - 장점: 현재 증상에 대한 최소 수정이 가능하다.
   - 단점: 공통 primitive 결함이면 다른 위치의 동일 버그는 남을 수 있다.

### 추천

- 먼저 재현 경로와 상태 소유 위치를 추적한 뒤, 원인이 해당 기능 플로우에 국한되면 선택지 2로 최소 수정한다.
- primitive 자체 결함이 코드 증거로 확인될 때만 선택지 1로 올린다.
