# 89. 스페이스 설정 허브/진행기간/수강생 정보 구성 분리

## 차수 1

### 작업내용

- 학생 목록의 `스페이스 설정` 진입을 단일 대형 편집 화면에서 `허브형 설정 진입`으로 재구성한다.
- 허브는 `진행기간`, `수강생 정보 구성` 두 버튼만 보이는 단순 버튼형으로 만든다.
- 모달 위에 다른 모달을 띄우지 않고, 같은 설정 셸 내부에서 `허브 -> 진행기간 -> 수강생 정보 구성`으로 화면이 전환되도록 만든다.

### 논의 필요

- 허브/진행기간/구성 편집 화면 사이 전환 시 크기 변화가 자연스럽게 보여야 한다.
- 진행기간 화면은 "작은 전용 모달처럼" 느껴져야 하지만 실제 구현은 같은 셸 내부 상태 전환으로 처리해야 한다.

### 선택지

- 선택지 1: 기존처럼 하나의 큰 화면 안에 진행기간과 구성을 함께 둔다.
- 선택지 2: 허브에서 각 화면으로 나누되, 중첩 모달로 올린다.
- 선택지 3: 허브에서 각 화면으로 나누고, 같은 셸 내부에서 뷰만 전환한다.

### 추천

- 선택지 3

### 사용자 방향

- 허브형
- 단순 버튼형 허브
- 진행기간은 작은 전용 모달처럼 보이되 같은 설정 셸 내부 전환
- 수강생 정보 구성은 기존 대형 편집 화면을 유지하되 진행기간 섹션 제거

## 차수 2

### 작업내용

- 학생 목록/모바일 액션의 `스페이스 설정` 버튼은 그대로 유지하고, 새 허브를 열도록 연결한다.
- 기존 `수강생 정보 구성` 화면 제목과 문구는 유지하되, 진행기간 관련 UI와 상태를 제거한다.
- 검증 후 현재 설정 흐름에서 회귀가 없는지 점검한다.

### 논의 필요

- 없음

### 선택지

- 선택지 1: 데스크톱만 허브로 바꾸고 모바일은 기존 흐름 유지
- 선택지 2: 데스크톱/모바일 모두 같은 허브 흐름으로 통일

### 추천

- 선택지 2

### 사용자 방향

- 데스크톱/모바일 모두 같은 허브 흐름
