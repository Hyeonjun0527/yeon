# 상담 분석 파이프라인 개선 백로그

## 배경

테스트 녹음(TTS 2인 대화) 업로드 결과:

- 화자분리 미동작 (전체가 단일 "원문" 세그먼트)
- AI 분석에서 학생정보/특징 미추출 (자유형 텍스트만 생성)
- `.env`에 `gpt-4o-transcribe-diarize` 설정되어 있으나 fallback 발생 추정

## 1차: AI 분석 프롬프트 구조화

### 작업내용

- `buildInitialAnalysisPrompt()` 강화: 학생정보, 학생특징, 감정/태도, 핵심이슈, 후속조치를 명시적으로 추출
- 시스템 프롬프트 용어 수정: "교사" → "멘토/운영자", "학생" → "수강생"
- 화자분리 없을 때도 AI가 대화 맥락에서 발화자를 추론하도록 프롬프트에 가이드 추가

### 논의 필요

- 구조화된 JSON 응답 vs 마크다운 응답 (현재 마크다운)

### 선택지

- A: 마크다운 유지하되 프롬프트를 정밀하게 (변경 최소)
- B: JSON 스키마 응답 강제 (프론트 파싱 필요)

### 추천

A — 현재 스트리밍 채팅 구조에서 JSON 강제는 과잉. 프롬프트 개선으로 충분.

## 2차: 화자분리 fallback 진단 및 로깅

### 작업내용

- `mapOpenAiSegments`에서 segments가 비어 fallback되는 경우 로그 추가
- `buildFallbackSegments` 진입 시 어떤 모델이 사용됐는지 기록

### 논의 필요

- 없음

### 선택지

- 단일 선택지: 로그 추가

### 추천

바로 구현

## 3차: 화자분리 품질 개선

### 작업내용

- `MAX_DIARIZE_CHUNK_DURATION_SECONDS = 30` 값이 너무 짧을 수 있음 → 검토
- 화자분리 실패 시 AI 후처리로 발화자 추론하는 2차 패스 검토

### 논의 필요

- 30초 청크가 화자분리 품질에 악영향을 주는지

### 선택지

- A: 청크 길이 늘리기 (60초~120초)
- B: 현재 유지 + AI 후처리 추가
- C: 청크 길이 + AI 후처리 모두

### 추천

A 우선 시도 — 30초는 대화 맥락 부족으로 화자 구분이 어려울 수 있음
