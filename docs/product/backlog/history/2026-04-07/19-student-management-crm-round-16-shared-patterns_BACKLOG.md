# 19-student-management-crm-round-16-shared-patterns_BACKLOG

## 현재 상태

- `/student-management` 화면에는 우선 학생 큐, 학생 상세, 상태 패널, 보조 패널이 모두 구현되어 있다.
- 하지만 배지, 상태 라벨, 섹션 헤더, 액션 버튼 패턴이 화면 내부에 중복 선언되어 있어 round 17 이후 코호트 전환 UI를 올릴 때 일관성 유지 비용이 커진다.
- 현재 `/` 워크스페이스는 다른 에이전트가 동시에 수정 중이라, 이번 차수는 `origin/develop` 기준으로 안정한 `student-management` 경로에서 먼저 패턴을 공용화하는 편이 안전하다.

## 이번 진행 차수

- `16차`

## 목표 상태

- 학생관리 대시보드와 학생 상세가 같은 배지, 상태 라벨, 액션 버튼, 섹션 헤더 패턴을 사용한다.
- 색상과 간격 규칙이 한 곳에 모여 다음 round에서 코호트 선택, 주차 컨텍스트, 추가 패널을 붙여도 디자인 시스템이 흔들리지 않는다.
- 카드 전체를 무리하게 공용화하지 않고, round 16 권장 범위인 배지/버튼/섹션 헤더만 추출한다.

## 16차

### 작업내용

- `student-management` 화면에서 공통으로 쓰는 배지, 섹션 헤더, 버튼 패턴을 앱 공용 UI 프리미티브로 추출한다.
- 위험도, 세그먼트, 학습 신호에 대응하는 상태 라벨 source of truth를 한 파일로 정리한다.
- 상태 패널과 우선 학생 카드, 학생 상세 패널이 같은 프리미티브를 사용하도록 리팩토링한다.

### 논의 필요

- 공통 패턴을 현재 feature 내부에 둘지, 앱 공용 UI 레이어로 올릴지

### 선택지

- A. 앱 공용 UI 프리미티브로 올리고 도메인 라벨 매핑은 앱 로컬 source of truth로 둔다
- B. 전부 feature 내부에 둔다

### 추천

- A. round 17 이후 다른 학생관리 화면에서도 재사용할 수 있고, 카드 전체까지 공용화하지 않아도 충분히 안전하다
