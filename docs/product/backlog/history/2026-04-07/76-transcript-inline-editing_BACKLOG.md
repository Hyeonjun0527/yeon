# 76. 원문 교정 편집 (Transcript Inline Editing)

## 예상 차수: 4차

## 현황

- STT 전사 결과는 DB에 `counseling_transcript_segments` 테이블로 저장
- 현재 원문은 읽기 전용. 교사가 오타·화자 라벨 오류를 수정할 수 없음
- STT 오류가 AI 분석 품질에 직접 영향 → 교정이 분석 정확도의 기반

## 목표

- 교사가 원문 뷰어에서 세그먼트 텍스트와 화자 라벨을 직접 수정
- 수정 내역이 DB에 반영되어 이후 AI 분석에도 정정된 원문이 사용됨

---

### 차수 1: 서버 — 세그먼트 수정 API

**작업내용**

- `PATCH /api/v1/counseling-records/[recordId]/segments/[segmentId]`
- body: `{ text?: string, speakerLabel?: string, speakerTone?: string }`
- 소유권 검증, 부분 업데이트
- `counseling_records.transcriptText`도 전체 재조합해서 갱신
- `counseling_records.updatedAt` 갱신
- api-contract에 request/response 스키마 추가

**논의 필요**: 없음. 단순 PATCH.

**추천**: 표준 PATCH, 변경된 필드만 업데이트.

### 차수 2: 클라이언트 — 인라인 편집 모드

**작업내용**

- 세그먼트 텍스트 클릭/더블클릭 → 인라인 textarea로 전환
- 화자 라벨 클릭 → 드롭다운 (교사/학생/보호자/기타)
- Enter로 저장, Esc로 취소
- 저장 시 PATCH API 호출
- 낙관적 업데이트: UI 즉시 반영, 실패 시 롤백
- 수정된 세그먼트에 "수정됨" 표시 (미세한 아이콘 또는 배경)

**논의 필요**

- 수정 표시를 어떤 형태로 할지 (아이콘 vs 배경색 vs 없음)

**선택지**

- A) 수정된 세그먼트 좌측에 연필 아이콘 표시
- B) 수정된 세그먼트 배경을 미세하게 다르게
- C) 표시 없음 (수정이 자연스럽게 녹아들도록)

**추천**: C) 표시 없음. 교정은 일상적 작업이라 시각적 노이즈가 되면 안 됨.

**사용자 방향**:

### 차수 3: 일괄 화자 라벨 변경

**작업내용**

- "화자 A를 모두 '교사'로 변경" 같은 일괄 변경 기능
- 전사 헤더 영역에 "화자 일괄 변경" 버튼
- 모달/드롭다운: 현재 화자 목록 → 변경할 라벨 선택
- 서버: `PATCH /api/v1/counseling-records/[recordId]/segments/bulk` (speakerLabel 기준 일괄)

**논의 필요**: 없음

**추천**: STT 화자 분리 결과가 "A", "B"로 오는 경우가 많으므로 이 기능이 실무에서 매우 자주 쓰일 것.

**사용자 방향**:

### 차수 4: 세그먼트 병합/분할

**작업내용**

- 인접한 두 세그먼트를 하나로 병합 (같은 화자가 끊겨서 들어온 경우)
- 하나의 세그먼트를 특정 위치에서 분할 (화자가 바뀌었는데 합쳐진 경우)
- 서버: `POST /api/v1/counseling-records/[recordId]/segments/merge`, `POST .../split`
- segmentIndex 재정렬 로직

**논의 필요**

- 병합/분할이 초기에 꼭 필요한지, 아니면 텍스트 수정만으로 충분한지

**선택지**

- A) 1~3차까지만 하고 4차는 사용자 피드백 후 결정
- B) 전부 구현

**추천**: A) 텍스트 + 화자 수정만으로도 80%는 커버됨. 병합/분할은 후순위.
