# 19-landing-demo-entry BACKLOG

## 차수 1

### 작업내용

- 랜딩페이지에 `데모 보기` CTA를 추가한다.
- 현재 인증/세션 구조를 기준으로 소셜 로그인 없이 평가자가 즉시 진입할 수 있는 데모 엔트리 플로우를 만든다.
- 가능하면 기존 인증 경계를 크게 흔들지 않고, 평가용 샘플 데이터/데모 모드가 분명히 보이도록 구현한다.

### 논의 필요

- 데모 보기 진입이 완전 읽기 전용인지, 일부 상호작용을 허용할지 결정이 필요하다.
- 데모 세션을 별도 쿠키/플래그로 처리할지, 기존 사용자/스페이스 구조에 묶인 샘플 계정 세션으로 처리할지 판단이 필요하다.

### 선택지

1. 랜딩에서 `데모 보기` 클릭 시 별도 데모 쿠키를 심고 평가용 라우트/데이터만 허용한다.
2. 랜딩에서 `데모 보기` 클릭 시 미리 세팅된 샘플 계정 세션을 발급한다.
3. 랜딩에서 데모 전용 공개 페이지로만 보내고 실제 앱 세션 진입은 막는다.

### 추천

- 1번 또는 2번 중 현재 인증 경계에 덜 침습적인 방식을 택한다. 핵심은 평가자가 즉시 핵심 화면으로 들어가되, 실제 연동 계정 흐름과 충돌하지 않게 만드는 것이다.
