# 기능 100선

`YEON`을 `교강사 중심 AI 학생관리 서비스`로 설계할 때 고려할 수 있는 기능 100가지를 정리한 문서다.

이 문서의 목적은 `지금 당장 다 만든다`가 아니라, 공모전 심사자와 팀 내부 모두가 `이 제품이 어디까지 확장될 수 있는지`를 한눈에 이해하게 만드는 데 있다.

## 정리 원칙

- 핵심 사용자는 `교강사`다.
- 직접 효용은 `수강생 관리 정확도와 속도 향상`으로 설명 가능해야 한다.
- AI는 장식이 아니라 `위험 해석`, `개입 추천`, `피드백 초안`, `요약 자동화`에 집중한다.
- 기능은 `MVP 핵심`, `운영 확장`, `후속 차별화`가 한 흐름으로 이어지도록 묶는다.

## 1. 교강사 메인 워크스페이스

1. 학생함 우선순위 보드: 오늘 가장 먼저 확인해야 할 학생을 상단에 고정한다.
2. 위험도 색상 배지: 학생별 긴급도를 한눈에 구분한다.
3. 수업 시작 전 체크리스트: 교강사가 수업 전에 확인할 항목을 묶어준다.
4. 반별 관리 필요 학생 수 요약: 어느 반이 더 불안정한지 빠르게 파악한다.
5. 미처리 학생 모아보기: 아직 피드백하지 않은 학생만 다시 볼 수 있다.
6. 오늘의 주요 위험 신호 요약: 과제 지연, 결석, 질문 없음 같은 신호를 상단에 보여준다.
7. 긴급 학생 핀 고정: 꼭 챙겨야 하는 학생을 메인 화면에 고정한다.
8. 최근 개입 후 재확인 대상 목록: 지난 개입 이후 다시 관찰해야 할 학생을 별도로 보여준다.
9. 교강사 메모 위젯: 당일 수업 전에 볼 개인 메모를 저장한다.
10. 수업 전 브리핑 카드: 오늘 반 분위기와 핵심 리스크를 짧게 요약한다.

## 2. 학생 위험 신호 수집

11. 출석 연속 결석 감지: 연속 결석 패턴을 자동으로 잡아낸다.
12. 과제 제출 지연 감지: 마감 대비 지연 학생을 자동 분류한다.
13. 질문 빈도 저하 감지: 질문이 급감한 학생을 신호로 잡는다.
14. 퀴즈 성과 하락 감지: 이전 대비 성취가 떨어진 학생을 분리한다.
15. 학습 페이지 체류 저하 감지: 콘텐츠 진입은 했지만 실제 학습 시간이 짧은 경우를 감지한다.
16. 실습 실패 반복 감지: 같은 과제나 테스트에서 반복 실패하는 패턴을 찾는다.
17. 피드백 미열람 감지: 교강사 피드백을 읽지 않은 학생을 식별한다.
18. 지각 빈도 상승 감지: 지각이 누적되는 학생을 위험군으로 본다.
19. 팀 활동 비참여 감지: 협업형 과제에서 존재감이 낮은 학생을 찾는다.
20. 휴면 학생 자동 태깅: 최근 활동이 거의 없는 학생을 자동 태그한다.

## 3. 데이터 해석과 AI 진단

21. 학생별 위험 원인 요약: 왜 위험하다고 판단했는지 문장으로 설명한다.
22. 다중 신호 종합 점수: 여러 신호를 하나의 위험도로 합산한다.
23. 신호별 기여도 표시: 어떤 데이터가 판단에 가장 크게 영향을 줬는지 보여준다.
24. 최근 2주 변화 추세 요약: 좋아지는지 나빠지는지 흐름을 설명한다.
25. 반 평균 대비 비교: 개인 문제가 반 전체 문제인지 구분한다.
26. 과거 유사 사례 매칭: 이전 학생 케이스와 유사한 패턴을 보여준다.
27. 수업 난도 영향 추정: 학생 문제인지 수업 난도 문제인지 해석을 돕는다.
28. 개입 타이밍 추천: 지금 즉시 개입해야 하는지, 하루 더 지켜봐도 되는지 제안한다.
29. 케어 우선순위 재정렬: 새로운 데이터가 들어오면 위험 순서를 자동 갱신한다.
30. 위험 오탐 피드백 학습: 교강사가 아니라고 표시한 사례를 다음 판단에 반영한다.

## 4. 피드백과 개입 실행

31. 1클릭 피드백 초안 생성: 학생 상황에 맞는 메시지를 바로 생성한다.
32. 톤 선택형 피드백 생성: 단호형, 공감형, 격려형 등 톤을 고를 수 있다.
33. 출석 문제 전용 피드백 템플릿: 결석/지각 상황에 맞는 기본 문구를 제공한다.
34. 과제 지연 전용 피드백 템플릿: 제출 독려 메시지를 빠르게 만든다.
35. 질문 유도 메시지 생성: 조용한 학생에게 질문을 열어주는 문구를 만든다.
36. 실습 보충 안내 초안: 보충 학습이나 재시도를 안내하는 문구를 만든다.
37. 학부모·보호자 공유용 요약 초안: 필요한 교육 환경에서 외부 공유 문안을 만든다.
38. 학생별 다음 행동 제안: 피드백 뒤 학생이 해야 할 액션을 정리한다.
39. 개입 결과 기록 버튼: 메시지 발송, 면담, 추가 과제 부여 등을 이력으로 남긴다.
40. 재개입 리마인드 예약: 며칠 뒤 다시 확인할 시점을 예약한다.

## 5. 학생 프로필과 케어 이력

41. 학생 케어 타임라인: 출석, 과제, 피드백, 면담 기록을 시간순으로 본다.
42. 학생 기본 프로필 카드: 목표, 반, 수강 시작일, 최근 상태를 한 카드에 보여준다.
43. 강점·약점 요약: 학습 성향과 반복 약점을 축적한다.
44. 최근 위험 원인 스냅샷: 현재 문제의 핵심을 요약해 보여준다.
45. 개입 전후 변화 비교: 개입 이후 신호가 개선됐는지 본다.
46. 학생 개인 메모: 교강사가 상황적 맥락을 기록한다.
47. 상담 이력 보관: 면담 내용과 후속 조치를 저장한다.
48. 학습 목표 추적: 학생이 약속한 목표와 현재 이행 정도를 비교한다.
49. 케어 태그 시스템: `집중 저하`, `과제 미루기`, `자신감 부족` 같은 태그를 붙인다.
50. 학생별 추천 개입 히스토리: 어떤 종류의 개입이 효과적이었는지 누적한다.

## 6. 수업·반 운영 기능

51. 반별 학생 상태 매트릭스: 전체 학생을 위험도와 참여도로 매핑한다.
52. 수업 전 반 요약 리포트: 오늘 수업에서 특히 챙겨야 할 학생군을 보여준다.
53. 주차별 핵심 이슈 정리: 특정 주차에 많이 흔들린 개념을 요약한다.
54. 반 분위기 체감 기록: 교강사가 수업 후 반 분위기를 메모한다.
55. 수업 난이도 표시 기능: 해당 수업 주차의 체감 난도를 기록한다.
56. 학생군 세그먼트 자동 분류: 안정권, 관찰권, 위험권으로 나눈다.
57. 보충 필요 학생 묶음 추천: 함께 보충하면 좋은 학생군을 추천한다.
58. 과제 재설계 힌트 제공: 다수 학생이 막힌 과제의 개선 포인트를 제안한다.
59. 수업 중 실시간 체크용 미니 패널: 수업 중 빠르게 학생 상태를 확인한다.
60. 반별 개입 커버리지 지표: 누가 케어를 받고 누가 놓였는지 보여준다.

## 7. 리포트와 회고 자동화

61. 주간 케어 리포트 자동 생성: 반 단위 주요 이슈와 개입 내역을 요약한다.
62. 학생 상태 변화 리포트: 지난주 대비 위험도 변화 학생을 보여준다.
63. 반 전체 위험 원인 분포: 어떤 문제가 가장 흔한지 집계한다.
64. 교강사 개입 활동 요약: 한 주 동안 어떤 대응을 했는지 정리한다.
65. 학습 부진 조기 경보 리포트: 탈락 위험 학생을 조기에 모아 보여준다.
66. 개입 효과 분석 리포트: 어떤 유형의 개입이 효과 있었는지 본다.
67. 반복 이슈 개념 리포트: 특정 개념에서 계속 막히는 학생군을 찾는다.
68. 운영 공유용 관리자 리포트: 교육 운영팀에 넘길 수 있는 형태로 요약한다.
69. 학기·코호트 비교 리포트: 기수별 학생 관리 패턴 차이를 비교한다.
70. 제출용 성과 스토리 자동 초안: 공모전 발표나 내부 보고용 스토리라인을 만든다.

## 8. 운영자·협업·권한

71. 교강사 계정별 반 접근 권한: 담당 반만 보이게 제한한다.
72. 운영자 전체 반 대시보드: 운영자는 여러 반을 한눈에 관리한다.
73. 케어 인계 메모: 교강사 교체 시 학생 맥락을 이어받는다.
74. 공동 코멘트 기능: 운영자와 교강사가 같은 학생에 코멘트를 남긴다.
75. 확인 완료 체크: 특정 리스크를 누가 확인했는지 남긴다.
76. 학생 이슈 에스컬레이션: 운영팀 개입이 필요한 학생을 상위 흐름으로 올린다.
77. 권한별 데이터 마스킹: 필요 없는 개인정보는 일부 숨긴다.
78. 교강사별 처리 속도 지표: 팀 운영 관점에서 대응 속도를 본다.
79. 반 오픈·종료 관리: 운영 주기 시작과 종료를 명확히 관리한다.
80. 운영 이슈 기록 보드: 학생 개인 문제 외의 반 운영 문제도 함께 기록한다.

## 9. 자동화·연동·알림

81. LMS 출석 데이터 자동 동기화: 기존 시스템과 출석 데이터를 연결한다.
82. 과제 제출 시스템 연동: 과제 제출 상태를 자동으로 가져온다.
83. 퀴즈·시험 결과 연동: 성취도 데이터를 자동 반영한다.
84. 슬랙 알림 연동: 긴급 학생 발생 시 교강사에게 즉시 알린다.
85. 이메일 발송 연동: 생성한 피드백을 바로 메일로 보낸다.
86. 캘린더 기반 리마인드: 개입 예정 일정을 달력과 연결한다.
87. CSV 업로드 기능: 초기 데이터가 흩어져 있을 때 수동 업로드를 지원한다.
88. 외부 메시지 템플릿 저장소: 반복 메시지를 재사용할 수 있게 만든다.
89. 주간 요약 자동 발송: 교강사와 운영팀에 리포트를 자동 배포한다.
90. 데이터 누락 감지 알림: 필요한 데이터가 들어오지 않았을 때 관리자에게 알린다.

## 10. 신뢰성·개인화·후속 확장

91. AI 판단 근거 공개 모드: 왜 이 판단이 나왔는지 더 자세히 볼 수 있다.
92. 교강사 맞춤 우선순위 조정: 교강사마다 중요하게 보는 신호 비중을 조절한다.
93. 위험도 기준 커스터마이징: 기관별 운영 기준에 맞게 임계값을 바꾼다.
94. 학생 유형별 케어 플레이북: 조용한 학생, 과제 회피형 학생 등 대응 가이드를 제공한다.
95. 개입 실험 기능: 서로 다른 피드백 방식을 시험해 효과를 본다.
96. AI 초안 승인 워크플로: 자동 발송 전 반드시 사람이 확인하게 한다.
97. 개인정보 접근 로그: 누가 어떤 학생 정보를 열람했는지 남긴다.
98. 설명 가능한 추천 이력: 추천이 어떻게 바뀌었는지 추적한다.
99. 모바일 교강사 뷰: 이동 중에도 긴급 학생만 빠르게 볼 수 있다.
100.  멀티 교육 유형 확장: 부트캠프 이후 대학, 사내 교육, 학원으로 확장 가능하게 설계한다.

## 문서 활용 가이드

- 공모전 제출용 핵심 스토리는 `1~5번`, `21~24번`, `31~41번`, `51~53번`, `61~68번`을 중심으로 잡는 것이 좋다.
- 실제 MVP 구현은 이 중에서도 `학생함`, `학생 상태 카드`, `피드백 초안`, `개입 이력`, `반 단위 리포트`를 우선한다.
- 나머지 기능은 심사자에게 `확장성`, `운영 가능성`, `AI 활용 깊이`를 보여주는 근거로 활용한다.
