# 07. 교강사 중심 MVP 사용자 흐름

## 기획 목표

이번 MVP의 목표는 "부트캠프 교강사가 전체 코호트 학생을 한 번에 보고, 필요한 학생을 찾아, 개입하고, 기록하고, 다시 확인하게 만든다"는 한 문장으로 정리한다. 분석 기능을 많이 붙이는 것보다, 교강사가 실제로 학생관리를 더 잘하게 되는 흐름을 짧고 명확하게 보여주는 것이 우선이다.

## 핵심 사용자

- 주 사용자: `운영 조직 소속 부트캠프 교강사`
- 직접 수혜자: `수강생`
- 보조 사용자: `교육 운영자`

## 대표 페르소나 상세

- `박서연`, 운영 조직 소속 메인 교강사
- 2개 코호트, 총 48~60명을 맡고 조교와 협업
- 수업 전 30분 안에 오늘 챙길 학생과 운영 리스크를 함께 정리해야 함
- 출석, 과제, 질문, 상담 메모가 각각 흩어져 있어 학생 상태를 다시 조합하는 데 시간이 많이 듦
- 메시지 초안 작성보다 `누구를 왜 먼저 챙겨야 하는지`를 정리하는 데 더 큰 피로를 느낌

## 교강사의 핵심 상황

- 코호트 학생 수가 많아질수록 누가 흔들리는지 놓치기 쉽다.
- 출석, 데일리 과제, 실습, 질문, 상담 메모를 각각 따로 보게 된다.
- 피드백을 해야 한다는 건 알지만, 누구부터 어떤 말로 시작할지 정리하는 데 시간이 든다.
- 이전에 어떤 학생에게 무엇을 했는지 기록이 흩어져 후속 관리가 끊긴다.
- 반 전체를 보면서도 개별 학생을 놓치지 않는 구조가 없다.
- 운영 조직에 공유해야 할 반 리스크와 학생관리 흐름이 분리되어 있다.

## 교강사의 하루 업무 흐름

### 수업 전

- 출석, 과제, 질문, 상담 메모를 다시 조합해 오늘 챙길 학생을 추린다.
- 반 전체 리스크와 운영 공유 포인트를 정리한다.
- MVP가 먼저 해결할 순간은 이 `수업 전 준비 시간`이다.

### 수업 중

- 실습 참여, 질문 반응, 현장 분위기를 보며 즉시 개입 대상을 확인한다.
- 다만 이 시점은 실시간 대응 성격이 강해, MVP 1차 우선순위보다 후순위로 둔다.

### 수업 후

- 보낸 피드백, 면담 메모, 보강 과제, 다음 확인일을 남긴다.
- 운영 조직과 공유할 주간 리스크를 정리한다.

## 위험 신호 우선순위

1. `참여 저하`
   출석 하락, 지각 증가, 실습 참여 급감은 가장 빠르게 개입해야 하는 조기 경고 신호다.

2. `데일리 과제 지연`
   지연 제출과 미제출 누적은 실습 흐름 단절과 직접 연결된다.

3. `질문 없음`
   질문이 줄어든 학생은 늦게 발견될수록 개입 비용이 커진다.

4. `상담 메모 경고`
   개인 사정, 자신감 저하, 반복 경고는 다른 신호와 함께 우선순위를 조정하게 만든다.

## MVP 한 줄 시나리오

`운영 조직 소속 부트캠프 교강사는 로그인 후 학생함에서 전체 코호트 학생 상태를 보고, 수업 전 30분 안에 관리가 필요한 학생을 선별하고, 학생별 상태 카드와 이력을 확인한 뒤, AI가 제안한 피드백 초안을 검토하고 실행하며, 다음 확인 일정과 운영 공유 포인트까지 이어서 관리한다.`

## 제출용 한 줄 설명

`YEON은 운영 조직 소속 부트캠프 교강사가 수업 전 30분 안에 오늘 챙길 학생을 정리하고 개입까지 이어가게 만드는 AI 학생관리 CRM입니다.`

## 핵심 화면 구조

### 1. 교강사 메인 대시보드

- 전체 학생 수와 세그먼트 요약
- 코호트 주차와 오늘 실습 진행 상태
- 오늘 확인할 후속관리 항목
- 가장 위험한 학생 3명
- 반복적으로 막히는 개념과 실습 구간
- 즉시 행동 버튼
  - `학생함 열기`
  - `피드백 보내기`
  - `면담 메모 남기기`
  - `보강 과제 지정`

### 2. 학생함과 관리 리스트

- 학생 이름
- 현재 상태
- 위험도 또는 우선순위
- 위험 이유 1줄 요약
- 최근 개입 여부
- 다음 확인일
- 최근 변화
  - 출석 하락
  - 데일리 과제 지연
  - 질문 없음
- 추천 행동 버튼

### 3. 학생 상세 관리 카드

- 학생 기본 상태 요약
- 최근 학습 신호 타임라인
- 최근 코드 리뷰/실습 피드백 요약
- 상담/면담 메모 요약
- AI 위험 해석
- 피드백 초안
- 교강사 수정 입력
- 발송/저장 버튼

### 4. 후속관리와 개입 이력 화면

- 언제 개입했는지
- 어떤 메시지를 보냈는지
- 후속 확인 필요 여부
- 다음 액션 종류
- 다음 수업 전 재확인 대상 표시

### 5. 반 단위 운영 리포트

- 이번 주 학생 관리 요약
- 아직 한 번도 개입하지 않은 학생군
- 반복적으로 막히는 개념과 실습 구간
- 아직 미개입 학생 수
- 다음 수업 준비 포인트
