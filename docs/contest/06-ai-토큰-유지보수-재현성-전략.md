# 06. 토큰 낭비 최소화 / 유지보수성 / 재현성 전략

이 프로젝트는 AI를 많이 쓰는 것보다 `같은 판단을 다시 하지 않게 만드는 구조`를 더 중요하게 본다. 이를 위해 먼저 backlog, 공모전 문서, 저장소 운영 메모를 파일로 남겨 다음 세션에서도 같은 문맥을 재현할 수 있게 한다. 즉, 긴 프롬프트를 매번 반복하는 대신 `경로 기반 컨텍스트`를 남기는 전략을 사용한다.

토큰 낭비를 줄이기 위한 실무 전략은 다음과 같다.

- 필요한 파일만 지연 로딩하고, 저장소 전체를 매번 다시 읽지 않는다.
- 문서 질문마다 Markdown 파일을 분리해 수정 범위를 국소화한다.
- source data와 API 계약을 따로 두어, 교강사 UX 설명과 API 응답이 다른 말로 어긋나지 않게 한다.
- 같은 의미를 여러 프롬프트에서 다시 설명하지 않고, `backlog -> domain data -> contract -> UI` 순서의 source of truth를 유지한다.

유지보수성과 재현성을 높이기 위한 전략은 다음과 같다.

- 실제 구현 전 backlog를 작성해 차수별 범위를 고정한다.
- 제출 문서와 MVP가 같은 교강사 중심 문제 정의를 공유하도록 데이터 구조를 통합한다.
- 검증은 가능한 한 고정된 순서로 수행한다. 이 저장소에서는 최소한 typecheck와 형식 검증 결과를 남긴다.
- 후속 차수에서 실측 데이터나 프롬프트 정책이 바뀌더라도, 수정 지점이 문서/공용 데이터/계약 파일로 분리되어 있어 영향 범위를 빠르게 추적할 수 있다.
