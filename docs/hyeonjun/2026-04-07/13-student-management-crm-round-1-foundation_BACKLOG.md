# 13-student-management-crm-round-1-foundation_BACKLOG

## 현재 상태

- `origin/develop` 기준 현재 저장소에는 공모전용 문서와 랜딩 구현이 아직 없다.
- 사용자 요구는 `교강사용 AI 학생관리 CRM` 방향으로 공모전용 기획과 웹 MVP를 정리하는 것이다.
- 이번 차수에서는 `3차`, `4차`, `5차` 확정값을 문서와 UI에 먼저 고정해야 다음 구현 차수에서 흔들리지 않는다.

## 이번 진행 차수

- `1차`

## 목표 상태

- `수업 전 / 수업 중 / 수업 후` 하루 업무 흐름이 문서와 랜딩에 반영되어 있다.
- `참여 저하 -> 과제 지연 -> 질문 없음 -> 상담 메모 경고` 위험 신호 우선순위가 고정되어 있다.
- `교강사용 AI 학생관리 CRM` 포지셔닝과 제출용 한 줄 설명, 문제 정의가 공용 source of truth로 정리되어 있다.
- 웹 랜딩과 공용 JSON 응답이 같은 값을 노출한다.

## 차수

## 1차

### 작업내용

- 공모전 문서 패키지와 랜딩 기본 뼈대를 현재 브랜치에 복구한다.
- `contest overview`와 `instructor dashboard` 읽기 계약, 모의 데이터, API route를 함께 연결한다.

### 논의 필요

- 이전 차수 전체를 다 복구할지, 3차~5차에 필요한 최소 세트만 만들지

### 선택지

- A. 최소 세트만 복구
- B. 전체 공모전 문서 패키지 복구

### 추천

- B. 심사와 데모에서 바로 재사용할 수 있게 공모전 문서 패키지 전체를 함께 복구한다

### 사용자 방향

- B. 전체 공모전 문서 패키지 복구

## 2차

### 작업내용

- 교강사의 하루 업무 흐름을 수업 전, 수업 중, 수업 후로 분해하고 MVP 우선 순간을 명시한다.

### 논의 필요

- MVP가 가장 먼저 줄여야 하는 교강사 부담이 어느 시점인지

### 선택지

- A. 수업 전 준비
- B. 수업 후 정리

### 추천

- A. 오늘 챙길 학생 선별 가치가 가장 직접적으로 드러난다

### 사용자 방향

- A. 수업 전 준비로 확정

## 3차

### 작업내용

- 위험 신호를 조기 개입 기준으로 우선순위화하고 설명 문구를 고정한다.

### 논의 필요

- 시험/퀴즈 성과보다 앞서 볼 신호를 어디까지 둘지

### 선택지

- A. 참여 저하 + 과제 지연 + 질문 없음 + 상담 메모
- B. 시험/퀴즈 성과 중심

### 추천

- A. 교강사 조기 개입 서비스에 더 적합하다

### 사용자 방향

- A. 참여 저하 + 과제 지연 + 질문 없음 + 상담 메모로 확정

## 4차

### 작업내용

- 제출용 한 줄 설명, 문제 정의, 포지셔닝을 source of truth로 만들고 웹에 노출한다.

### 논의 필요

- `서비스`보다 `CRM` 표현을 전면에 둘지

### 선택지

- A. 교강사용 AI 학생관리 CRM
- B. 교강사용 AI 학생관리 서비스

### 추천

- A. 운영 도구와 관리 흐름의 성격이 더 선명하다

### 사용자 방향

- A. 교강사용 AI 학생관리 CRM으로 확정

## 5차

### 작업내용

- 문서, 공용 계약, 랜딩 UI가 같은 값으로 정렬되도록 검증하고 배포 전 확인한다.

### 논의 필요

- 문서 중심으로 끝낼지, 웹 랜딩까지 같이 노출할지

### 선택지

- A. 문서 + 웹 랜딩 + 공용 JSON까지 함께 반영
- B. 문서만 반영

### 추천

- A. 심사와 데모에서 한 번에 확인 가능하다

### 사용자 방향

- A. 문서 + 웹 랜딩 + 공용 JSON까지 함께 반영
