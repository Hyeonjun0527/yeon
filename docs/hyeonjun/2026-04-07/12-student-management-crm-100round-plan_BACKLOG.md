# 12-student-management-crm-100round-plan_BACKLOG

## 현재 상태

- 저장소는 Next.js 웹과 공용 패키지 뼈대만 있는 초기 스캐폴드 상태다.
- 공모전용 문제 정의와 기능 100선 문서는 아직 제품 홈, API 계약, 데모 데이터로 연결되지 않았다.
- 사용자가 요구한 `라운드 1~100` 형식의 실행 계획 문서는 아직 없다.

## 목표 상태

- `교강사 중심 AI 학생관리 CRM`을 100개 라운드로 순차 실행 가능한 backlog로 고정한다.
- 각 라운드는 한 번의 프롬프트 또는 짧은 구현 묶음으로 끝낼 수 있는 크기로 유지한다.
- 제품, 문서, 계약, 데이터, 배포, 검증이 같은 라운드 체계 안에서 추적되게 만든다.

## round 운영 메모

- 이 문서는 현재 저장소의 canonical round 1~100 계획 문서로 사용한다.
- 사용자가 앞선 라운드를 건너뛰어 지시하더라도, 실제 진행은 아직 완료되지 않은 가장 이른 라운드부터 수행한다.
- 별도 `사용자 방향`이 비어 있으면 `추천` 기준으로 진행한다.

## 구현 원칙

- 라운드 순서는 건너뛰지 않는다.
- 먼저 `source of truth`가 되는 계약과 데이터 구조를 고정한 뒤 UI와 상호작용을 얹는다.
- 홈 첫 인상은 설명형 랜딩보다 `교강사가 실제로 무엇을 할 수 있는가`를 우선한다.
- 인증, 실제 AI 호출, 실제 DB는 라운드 순서상 필요 시점까지 미루고, 그 전에는 데모 완결성과 정합성을 우선한다.
- 같은 의미의 구조는 문서, API 계약, 로컬 mock data, UI가 같은 언어를 보도록 맞춘다.
