## 1차

### 작업내용

- 수강생 스프레드시트 import 파이프라인에서 이름/이메일/전화번호 중심으로 잘려 나가던 추가 컬럼을 유지한다.
- 대용량 파일 분석 시 추가 컬럼을 `customFields`로 보존하고, 미리보기 UI에서도 동적으로 노출/수정 가능하게 만든다.
- import 시 새 스페이스의 기본 시스템 탭과 overview 커스텀 필드를 함께 만들고, 각 수강생의 추가 컬럼 값을 저장한다.

### 논의 필요

- 추가 컬럼 타입을 얼마나 공격적으로 추론할지
- 미리보기 테이블에서 너무 많은 컬럼이 생겼을 때 가독성을 어떻게 유지할지

### 선택지

- A. 모든 추가 컬럼을 text로 저장
- B. 헤더/값 패턴으로 email/phone/url/date/long_text 정도만 얕게 추론
- C. 별도 컬럼 매핑 UI까지 추가

### 추천

- 이번 차수는 B로 간다. 구조 변경은 작게 유지하면서도 GitHub/포트폴리오/생년월일 같은 대표 컬럼은 더 자연스럽게 저장할 수 있다.
