# Student tab 수강이력 제거 백로그

## 차수 1

### 작업내용

- 수강생 상세 기본 탭 정의에서 `수강이력`을 제거한다.
- 스페이스 생성, 탭 reset, 템플릿 적용, import 생성 시 심어지는 기본 시스템 탭 세트도 동일하게 맞춘다.
- 학생 상세 화면의 legacy fallback과 동적 탭 렌더링에서 `수강이력` 전용 분기를 제거하고, `상담기록` 흐름은 유지한다.

### 논의 필요

- `비상연락처`도 개요로 흡수할지 여부는 이번 변경과 분리해 후속으로 판단하는 편이 안전하다.
- 이미 생성된 기존 스페이스의 `수강이력` 탭을 migration처럼 자동 정리할지, 앞으로 생성/reset/import되는 기본값만 바꿀지 정책이 필요하다.

### 선택지

1. 새로 생성되는 기본 탭만 바꾸고, 기존 UI fallback은 그대로 둔다.
2. 기본 탭 seed와 UI fallback/분기까지 함께 정리해 앞으로의 기본 경험을 완전히 일치시킨다.

### 추천

- 선택지 2
- 현재 문제는 단순 라벨 삭제가 아니라 정보 구조의 일관성 문제다. source of truth와 fallback UI가 어긋나면 reset/import/create마다 다시 나타날 수 있으므로, 기본값과 렌더 분기를 한 번에 같이 정리하는 편이 안전하다.
