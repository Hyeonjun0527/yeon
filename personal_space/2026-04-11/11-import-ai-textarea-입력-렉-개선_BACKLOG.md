# 가져오기 AI 요청 textarea 입력 렉 개선 백로그

## 차수 1

### 작업내용

- `apps/web/src/features/cloud-import` 가져오기 워크스페이스에서 AI 요청 textarea 입력 시 발생하는 버벅임의 원인을 추적한다.
- 입력 중 불필요한 대규모 리렌더 또는 무거운 계산이 있는지 확인하고, 가장 작은 안전한 구조 변경으로 타이핑 반응성을 개선한다.
- 기존 가져오기 UX, AI 요청 전송 동작, 미리보기/수정 기능은 유지한다.

### 논의 필요

- textarea만 국소적으로 분리해도 충분한지, 아니면 우측 패널 전체를 memoization 기준으로 다시 나눌지 판단이 필요하다.
- 현재 분석 결과 테이블 크기가 더 커질 가능성을 고려해 입력 UX 최적화를 import workspace 공통 패턴으로 승격할지 검토가 필요하다.

### 선택지

1. textarea와 채팅 입력 상태만 지역화하고, 무거운 테이블/패널 리렌더를 차단한다.
2. 우측 패널 전체를 더 큰 단위로 쪼개고 virtualization 또는 deferred state까지 도입한다.

### 추천

- 선택지 1
- 현재 증상은 입력 한 글자마다 큰 subtree가 같이 다시 그려지는 쪽일 가능성이 높다. 먼저 입력 상태와 무거운 렌더 트리를 분리해 타이핑 응답성을 회복하는 것이 영향 범위가 작고 빠르다.
