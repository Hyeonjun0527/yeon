# Student tab 비상연락처 개요 흡수 백로그

## 차수 1

### 작업내용

- 수강생 상세 기본 탭에서 `비상연락처` 시스템 탭을 제거한다.
- 스페이스 생성, reset, import, 템플릿 적용 시 심어지는 기본 시스템 탭 세트도 동일하게 정리한다.
- 기존 공간에 남아 있을 수 있는 `guardian` 시스템 탭은 상세 화면에서 개요로 흡수하거나 숨겨서, 사용자가 별도 탭을 보지 않도록 한다.
- 레거시 fallback UI가 비상연락처 데이터를 따로 보여주고 있다면 개요 쪽으로 옮겨 정보 구조를 맞춘다.

### 논의 필요

- 기존 스페이스에서 `guardian` 탭 아래 직접 추가한 커스텀 필드를 overview와 어떻게 합칠지 정책 결정이 필요하다.
- space settings에서도 guardian 탭을 완전히 숨길지, 기존 데이터 정리용으로 임시 노출할지 판단이 필요하다.

### 선택지

1. guardian 탭 라벨만 숨기고 실제 필드/데이터 구조는 그대로 둔다.
2. guardian 기본 탭을 제거하고, 필요한 경우 overview에서 guardian 데이터를 함께 읽어 정보 구조를 실제로 정리한다.

### 추천

- 선택지 2
- 이번 목적은 탭 수를 줄이는 게 아니라 정보 구조를 제품 방향에 맞게 정리하는 것이다. label만 숨기면 source of truth와 실제 노출 구조가 계속 어긋나므로, 기본 탭 seed와 overview 노출 경계를 같이 정리하는 편이 안전하다.
