# 학생관리 기능 기획 백로그

> 작성일: 2026-04-09
> 상태: Deep Interview 완료 → 구현 대기
> Deep Interview Spec: `.omc/specs/deep-interview-student-management.md`

## Deep Interview 결과 요약

- **최종 모호도**: 18% (threshold 20% 통과)
- **핵심 발견**: 상담 녹음 → AI STT 추출 → 학생 자동 등록 (기존 백로그와 정반대 흐름)
- **1차 MVP**: 서버 연결 없이 UI/UX만 mock data로 구현
- **타겟 규모**: 중규모 학원 (30~100명)
- **사용자 방향 확정 사항**:
  - 자동 등록(상담→AI) 우선, 수동 등록 보조
  - AI 추출 → 자동 등록 + 나중에 수정
  - 드래그/다중선택 방식 반배정
  - 1차는 디자인만 (mock data)

## 배경

YEON은 사설 학원/아카데미용 학생 관리 + 녹음 AI 요약 플랫폼이다.
현재 `counselingRecords.studentName`이 단순 문자열로 저장되어 있어, 학생 엔티티가 독립적으로 존재하지 않는다.
학생관리 기능을 통해 학생을 독립 엔티티로 승격하고, 상담 기록과 연결하며, 학원 운영에 필요한 관리 기능을 제공한다.

## 1차: 데이터 모델 및 핵심 엔티티 설계

### 작업내용

학생관리에 필요한 핵심 엔티티와 관계를 정의한다.

#### 핵심 엔티티

| 엔티티 | 설명 | 주요 필드 |
|--------|------|-----------|
| **students** | 학생 프로필 | id, name, grade, school, birthDate, gender, phone, memo, status(active/inactive/graduated), enrolledAt, avatarUrl |
| **student_guardians** | 학부모/보호자 | id, studentId, name, relationship, phone, email, isPrimary |
| **classes** | 반/수업 그룹 | id, name, subject, instructorName, schedule, capacity, year, semester, status |
| **class_enrollments** | 수강 관계 (N:M) | id, studentId, classId, enrolledAt, droppedAt, status |
| **student_tags** | 학생 태그 (유연한 분류) | id, name, color, createdByUserId |
| **student_tag_assignments** | 태그-학생 연결 | studentId, tagId |

#### 기존 테이블 영향

- `counseling_records.studentName` → `counseling_records.student_id` (FK) 마이그레이션 필요
- 기존 문자열 학생명을 students 테이블로 마이그레이션하는 데이터 변환 스크립트 필요

### 논의 필요

1. 학생 상태를 `active/inactive/graduated` 3가지로 충분한가, 아니면 `휴원(paused)` 등 추가 필요?
2. 년도별 관리를 `classes.year` 기준으로 할지, 별도 `academic_year` 엔티티를 둘지?
3. 학부모 연락처를 별도 테이블로 분리할지, 학생 테이블에 내장할지?

### 선택지

| # | 선택지 | 장점 | 단점 |
|---|--------|------|------|
| A | 학부모를 학생 테이블 필드로 내장 | 단순함, 빠른 구현 | 보호자 2명 이상 대응 어려움 |
| B | 학부모를 별도 테이블로 분리 | 유연함, 다수 보호자 지원 | 구현 복잡도 증가 |

### 추천

**B** — 학원 맥락에서 부모님 2명(엄마/아빠) 모두 연락처를 관리하는 경우가 흔하다. 별도 테이블이 실무적으로 맞다.

### 2-1. 학생 목록 (Student List) — `/student-management`

**목적**: 전체 학생을 한눈에 파악하고, 빠르게 검색·필터링·정렬한다.

**레이아웃**:
```
┌─────────────────────────────────────────────────────┐
│  학생 관리                              [+ 학생 등록] │
├─────────────────────────────────────────────────────┤
│  🔍 검색...          [반 ▾] [상태 ▾] [태그 ▾] [년도 ▾]│
├─────────────────────────────────────────────────────┤
│  ☐ 전체 선택  (32명)              [카드뷰 | 테이블뷰] │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │ 아바타    │ │ 아바타    │ │ 아바타    │           │
│  │ 김민준    │ │ 이서윤    │ │ 박지호    │           │
│  │ 고2 · 수학│ │ 중3 · 영어│ │ 고1 · 수학│           │
│  │ #우수 #장학│ │ #신규     │ │ #주의     │           │
│  │ 상담 3건  │ │ 상담 1건  │ │ 상담 0건  │           │
│  └──────────┘ └──────────┘ └──────────┘           │
│                                                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐           │
│  │ ...       │ │ ...       │ │ ...       │           │
│  └──────────┘ └──────────┘ └──────────┘           │
│                                                     │
│               [1] [2] [3] ... [>>]                   │
└─────────────────────────────────────────────────────┘
```

**핵심 인터랙션**:
- **검색**: 이름, 학교, 메모 등 통합 검색 (debounced)
- **필터**: 반, 상태(재원/퇴원/졸업), 태그, 년도별 다중 필터
- **뷰 전환**: 카드뷰(기본) ↔ 테이블뷰 토글
- **벌크 액션**: 체크박스 선택 → 반 배정, 태그 추가, 상태 변경
- **빈 상태**: 학생이 없을 때 일러스트 + "첫 학생을 등록해보세요" CTA

**카드 정보 위계**:
1. 아바타 + 이름 (가장 큰 글씨)
2. 학년 · 대표 과목/반
3. 태그 뱃지 (컬러)
4. 상담 기록 수 (기존 시스템 연결)

| 탭 | 내용 |

│  └──────────────────────────────────────────────┘  │

- 정원 시각화: 프로그레스 바로 현재/최대 인원 표시

### 2-5. 대시보드 위젯 (기존 대시보드 통합)

**목적**: 메인 대시보드에서 학생 현황을 빠르게 파악한다.

**위젯 내용**:
- 전체 재원생 수 / 이번달 신규 등록 수
- 상태별 분포 (재원/휴원/퇴원) 도넛 차트

- 상담 기록 없는 학생 알림 ("2주 이상 상담 없는 학생 5명")

## 3차: 상담기록 연결 마이그레이션

### 작업내용

- `counseling_records.student_id` FK 컬럼 추가

1. 기존 상담 기록의 `studentName`과 매칭되는 학생이 없으면 자동 생성할지?

## UI/UX 디자인 원칙

### 학원 맥락 특화

1. **빠른 접근**: 학원 강사는 수업 전후 짧은 시간에 확인한다. 3클릭 이내로 원하는 학생 정보에 도달해야 한다.
2. **년도 전환 용이**: 매년 3월 신학기에 대규모 학생 변동이 있다. 년도별 아카이브와 신년도 세팅이 매끄러워야 한다.
3. **모바일 우선 고려**: 학원 강사는 태블릿/폰으로도 자주 확인한다. 반응형 필수.
4. **벌크 작업**: 반 배정, 태그 변경 등 다수 학생을 한 번에 처리하는 흐름이 필수.

### 시각 디자인

1. **깔끔한 카드 UI**: 둥근 모서리, 적절한 그림자, 넉넉한 여백
2. **컬러 태그 시스템**: 학생 분류를 시각적으로 빠르게 구분
3. **아바타 이니셜**: 이름 첫 글자 + 파스텔 배경색 자동 생성
4. **상태 뱃지**: 재원(초록), 휴원(노랑), 퇴원(회색), 졸업(파랑)
5. **마이크로 인터랙션**: 카드 hover, 탭 전환, Sheet 슬라이드 등 Framer Motion 활용

### 기존 시스템 일관성

- 상담 기록 워크스페이스와 동일한 사이드바/GNB 구조 유지
- CSS Modules + Tailwind 혼합 스타일 유지
- Lucide 아이콘 일관 사용
