# 카카오 앱 분리 계정 인증 마이그레이션 백로그

## 1차

### 작업내용

- `users.email` 단일 유니크 제약을 제거해 같은 이메일의 별도 user 생성을 허용한다.
- 소셜 로그인 업서트 로직을 수정해 `같은 provider + 다른 providerUserId`는 기존 user에 강제로 붙이지 않고 새 user를 생성하게 바꾼다.
- 같은 이메일 user가 여러 개일 때는 기존 user를 임의로 고르지 않고 새 user를 생성하도록 인증 정책을 정리한다.
- 인증/스키마 테스트와 DB 마이그레이션을 추가한다.

### 논의 필요

- 일반 수동 사용자 생성 API는 여전히 이메일 중복을 막을지 여부

### 선택지

- 선택지 A: 로그인/수동 생성 모두 이메일 중복 허용
- 선택지 B: 소셜 로그인 분리만 허용하고 수동 생성 API는 기존처럼 이메일 중복 차단

### 추천

- 선택지 B
- 이번 요구는 카카오 앱 분리 로그인 대응이 핵심이므로, 일반 사용자 생성 정책까지 한 번에 넓히지 않는 편이 회귀가 적다.
