# 6-수강생카드-우클릭삭제-시프트선택안정화 BACKLOG

## 차수 1

### 작업내용

- 수강생 카드 목록에 우클릭 컨텍스트 삭제 흐름을 추가한다.
- 선택 모드 진입 규칙을 `Ctrl/Cmd+클릭` 또는 우클릭 기준으로 정리한다.
- 선택 모드가 아닐 때 `Shift+좌클릭`이 드래그처럼 느껴지는 문제를 막고 상세 열람으로 일관되게 보정한다.
- 기존 bulk delete API를 재사용해 1명 삭제와 다중 삭제를 같은 경계에서 처리한다.

### 논의 필요

- 우클릭 메뉴에 삭제 외 추가 액션을 지금 같이 넣을지 여부는 후속 판단이 필요하다.
- 선택 모드 시각 피드백이 현재 배지/상단 액션바만으로 충분한지 추후 UX 점검이 필요하다.

### 선택지

1. 우클릭 삭제를 별도 단건 API로 분기한다.
2. 현재 bulk delete API를 재사용해 단건/다건 삭제를 통합한다.
3. `Shift+좌클릭`을 항상 범위 선택으로 유지하고 브라우저 드래그만 막는다.

### 추천

- 2번으로 간다. 이미 bulk delete API와 선택 상태가 있으므로 source of truth를 늘리지 않고 단건/다건 삭제를 같은 삭제 경계로 묶는 편이 안전하다.
- `Shift+좌클릭`은 선택 모드일 때만 범위 선택으로 허용하고, 선택 모드가 아니면 상세 열람으로 처리해 현재 좌클릭 탐색 UX를 유지한다.
