# 4. 학생보드 연간 잔디 연도 페이징 BACKLOG

## 차수 1. 연간 잔디 기준과 연도 전환 source of truth 정리

- 작업내용
  - 학생 보드 잔디를 `스페이스 시작일이 속한 해`를 기본 진입 연도로 여는 연간 뷰로 바꾼다.
  - 스페이스가 여러 해를 걸치면 연도별로 페이지를 넘기듯 이동할 수 있게 한다.
  - 잔디는 `주간 정렬`을 유지하되, 선택한 연도 밖 패딩 칸과 스페이스 범위 밖 날짜는 ghost tone으로 유지한다.
- 논의 필요
  - 없음. deep-interview에서 사용자 결정 완료.
- 선택지
  - 선택지 A: 정확히 365칸만 보이는 일자 연속 그리드
  - 선택지 B: GitHub 감각의 주간 정렬 그리드
- 추천
  - 선택지 B. 기존 시각 문법을 가장 덜 깨고 월 헤더도 자연스럽게 유지된다.
- 사용자 방향
  - 교차연도는 페이지처럼 넘긴다.
  - 기본 진입 연도는 스페이스 시작일의 해다.
  - 잔디는 주간 정렬로 유지한다.

## 차수 2. 불필요 설명 제거와 UI 단순화

- 작업내용
  - `빠른 수정은 그대로 두고...` 문구를 제거한다.
  - `진행기간 전체 · 하루 2분할 · 범위 밖 날짜는 ghost cell로 유지` 문구를 제거한다.
  - 상단의 `이름 · 출석 · 과제 완료` + 빈 잔디 미리보기 블록을 제거한다.
  - 대신 연도 네비게이터와 실제 학생 행만 남겨 보드의 밀도를 높인다.
- 논의 필요
  - 없음. 사용자 요청이 명시적이다.
- 선택지
  - 선택지 A: 설명 문구를 더 짧게 축약
  - 선택지 B: 설명 문구를 제거하고 조작 UI만 남김
- 추천
  - 선택지 B. 현재 화면은 이미 의미가 충분하고, 설명 카드가 오히려 노이즈다.
- 사용자 방향
  - 설명 문구와 불필요 블럭 제거

## 차수 3. 회귀 방지 테스트와 검증

- 작업내용
  - 연도 목록 계산, 기본 진입 연도, 주간 정렬 연간 캘린더 계산을 helper 단위 테스트로 추가한다.
  - `lint`, `typecheck`, 관련 테스트를 실행해 green 상태를 확인한다.
- 논의 필요
  - 없음
- 선택지
  - 선택지 A: UI 컴포넌트 테스트 중심
  - 선택지 B: helper 테스트 중심
- 추천
