# 수강생 상세 상담기록 탭 녹음 진입 흐름 BACKLOG

## 결정 요약

- 수강생 상세 화면의 `상담기록` 탭에서 녹음 진입 버튼을 제공한다.
- 연결된 상담 기록이 없으면 빈 상태에 `녹음하기` 버튼을 노출한다.
- 연결된 상담 기록이 하나 이상 있으면 `녹음하기` 대신 `새 녹음 추가` 버튼을 노출한다.
- 버튼은 학생 상세 화면 안에서 모달을 띄우지 않고 즉시 상담관리 `/home`으로 이동시킨다.
- `/home`으로 이동한 뒤 기존 `새 상담 기록` 모달이 자동으로 열린다.
- 모달의 `바로 녹음하기`, `파일 업로드`, `텍스트로 기록하기`는 모두 기존 흐름을 그대로 사용한다.
- 이 진입에서는 현재 수강생의 `memberId`, `studentName`이 기본 컨텍스트로 자동 주입되어, 생성된 기록이 바로 해당 수강생에 연결된다.

## 1차

### 작업내용

- 수강생 상세 `상담기록` 탭 빈 상태/목록 상태에 CTA 버튼 위치와 문구 추가
- 기존 안내문을 버튼 중심 흐름으로 재정리
- 버튼 노출 규칙을 `recordCount === 0` / `recordCount > 0` 기준으로 고정
- `새 녹음 추가` 버튼은 탭 상단 헤더가 아닌 본문 상단에 배치

### 논의 필요

- 없음

### 선택지

- 없음

### 추천

- 빈 상태는 본문 중앙 CTA, 기록 존재 상태는 탭 상단 우측 CTA

### 사용자 방향

## 2차

### 작업내용

- 학생 상세 화면에서 `/home` 진입용 CTA만 두고, 모달은 `/home`에서 자동 오픈
- `바로 녹음하기`, `파일 업로드`, `텍스트로 기록하기`는 `/home`의 기존 모달에서 그대로 제공
- 현재 수강생의 `memberId`, `studentName`을 `/home` 진입 시 전달할 route state/query 규칙 정의

### 논의 필요

- `/home`에서 모달 자동 오픈을 어떤 URL/state로 받을지

### 선택지

- query param 기반
- route-state helper 기반

### 추천

- 기존 `memberId` / `spaceId` query 흐름을 유지하면서 `newRecordEntry`와 `studentName`만 일회성 query로 사용
- `/home` 진입 후 자동으로 기존 `useRecordEntry` 모달을 열고, URL 정리까지 한 번에 처리

### 사용자 방향
