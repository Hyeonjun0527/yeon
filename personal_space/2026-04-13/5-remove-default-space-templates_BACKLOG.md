## 차수 1

### 작업내용

- 기본/시스템 스페이스 템플릿을 API 응답과 실제 저장소에서 제거한다.
- 사용자 정의 템플릿 생성/저장/적용 흐름은 유지한다.
- 템플릿 관련 테스트 기대값을 새 정책에 맞게 정리한다.
- 타입체크와 빌드로 회귀를 확인한다.

### 논의 필요

- 기존 DB에 이미 저장된 시스템 템플릿도 즉시 정리할지 여부.
- 향후 기본 템플릿을 완전히 폐기할지, 숨김만 유지할지 여부.

### 선택지

1. 목록에서만 숨긴다.
2. 목록에서 숨기고 기존 시스템 템플릿 레코드도 정리한다.
3. 템플릿 기능 전체를 제거한다.

### 추천

- 사용자 요청대로 기본 템플릿은 즉시 보이지 않게 하고, 기존 시스템 레코드도 정리하는 2번이 가장 명확하다.
