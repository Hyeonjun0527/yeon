# 04. AI 도구와 모델 선정 이유

이 프로젝트는 `단일 주도구 + 보조 검토 도구` 구조를 채택한다. 핵심 구현과 구조 설계는 `Codex + GPT-5.4`를 중심으로 진행하고, 반복 탐색과 초안 정리는 `GPT-5.4-mini`를 활용한다. 이렇게 역할을 분리한 이유는 교강사의 실제 학생 관리 흐름을 정확하게 제품 구조로 옮기는 작업과, 반복 초안 작업을 분리해 토큰 낭비를 줄이기 위해서다.

보조적으로는 `Claude Code`를 장문 명세 검토와 반례 탐색에 활용할 수 있다. 즉, 실제 코드와 문서의 source of truth는 Codex 작업 결과로 통일하고, Claude Code는 "이 설계가 빠뜨린 예외가 없는가"를 점검하는 리뷰어 성격으로 배치한다. 필요 시 `Cursor`는 IDE 안에서 국소적인 탐색과 인라인 수정 보조 도구로만 제한적으로 사용한다.

정리하면, 이 조합의 선택 이유는 다음과 같다.

- `GPT-5.4`: 구조 설계, 교강사 중심 UX 설계, 공용 계약 정리처럼 정확도가 필요한 작업에 적합
- `GPT-5.4-mini`: 반복적인 탐색, 피드백 초안 구조 정리, 문서 초안 정리에 적합
- `Claude Code`: 장문 리뷰와 예외 케이스 점검에 강점
- `Cursor`: 편집기 내부의 빠른 맥락 확인에 유리하지만, 최종 source of truth로 두지 않음
