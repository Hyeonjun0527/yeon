# 05. AI 도구별 활용 전략

## 1) Codex + GPT-5.4

- 페르소나: `설계와 구현을 동시에 책임지는 메인 엔지니어`
- 핵심 전략: backlog와 저장소 규칙을 먼저 읽고, 교강사의 실제 학생 관리 흐름을 문제 정의 -> source data/계약 -> UI/API 구현 순서로 반영한다.
- 제약 사항: 존재하지 않는 API나 구조를 추측하지 않고, 저장소 안의 실제 파일과 스크립트 기준으로만 작업한다.
- 에이전트 구성 방식: 한 번의 큰 프롬프트로 끝내기보다, 문서화/공용 계약/웹 구현/검증을 차수로 나눠 처리한다.
- 데이터 흐름: contest 문서 -> 앱 로컬 source data(`apps/web/src/lib/contest-overview.ts`) -> 공용 계약(`packages/api-contract`) -> 웹/API 소비 구조로 연결한다.

## 2) Codex + GPT-5.4-mini

- 페르소나: `보조 탐색자와 초안 정리 담당`
- 핵심 전략: 반복되는 검색, 요약, 초안 정리를 빠르게 처리하고, 최종 결정은 GPT-5.4 단계에서 확정한다.
- 제약 사항: 최종 계약 결정, 구조 변경, 중요한 정책 판단은 단독으로 확정하지 않는다.
- 활용 지점: 문서 섹션 초안, 피드백 문구 구조 정리, 레이블 대안, 비핵심 반복 작업

## 3) Claude Code

- 페르소나: `반례를 찾는 리뷰어`
- 핵심 전략: 구현 완료 후 상태 정합성, 학생 관리 누락 가능성, 과한 자동화, 누락된 사용자 영향을 검토한다.
- 제약 사항: 최종 구현 기준과 공용 source of truth는 저장소 내 문서/계약 파일로 고정한다.
- 활용 지점: PR 전 리뷰, 기획 문서 논리 점검, 놓치기 쉬운 edge case 확인

## 4) Cursor

- 페르소나: `IDE 내부의 국소 탐색 보조자`
- 핵심 전략: 특정 파일 주변 탐색, 인라인 수정 후보 확인, 빠른 코드 읽기에 제한적으로 사용한다.
- 제약 사항: 장기 메모리나 최종 설계 근거로 사용하지 않고, 저장소 문서와 backlog를 우선한다.
