# landing login modal performance stabilization backlog

## 작업내용

### 1차

- 랜딩 페이지 CTA 클릭 경로를 기준으로 렌더 경계를 다시 나눈다.
- 로그인 모달을 랜딩 hero 트리와 분리해 포털 렌더링 구조로 옮긴다.

### 2차

- 로그인 모달 오픈 시 3D hero와 blur 합성 비용이 겹치지 않도록 시각 효과를 낮춘다.
- backdrop blur 제거 또는 축소, hero 상태 고정/비활성화로 클릭 직후 메인 스레드 부담을 줄인다.

### 3차

- 랜딩의 무거운 애니메이션이 modal open state와 불필요하게 함께 렌더되지 않도록 메모이제이션/분리를 적용한다.
- 로그인 CTA 클릭 후 모달이 뜨는 체감 지연을 줄인다.

### 4차

- Playwright로 비로그인 상태 랜딩 페이지에서 CTA 클릭 후 모달 표시를 검증한다.
- 타입체크로 회귀를 점검한다.

## 논의 필요

- 모달이 열릴 때 Spline hero를 완전히 숨길지, 정지된 상태로만 남길지.
- 랜딩 애니메이션 품질과 즉시 반응성 중 어느 쪽을 더 우선할지.

## 선택지

1. 최소 변경
   - blur 제거
   - modal 포털화
   - hero 메모이제이션
2. 강한 최적화
   - modal open 중 hero 비활성화/숨김
   - 랜딩 motion 일부 축소

## 추천

- 이번 차수는 **강한 최적화**로 진행한다.
- CTA 반응성을 우선하고, modal open 동안 hero 합성 비용을 줄이는 방향으로 구현한다.

## 사용자 방향

- 랜딩 로그인 CTA 렉을 구조적으로 줄이는 방향으로 구현 진행
