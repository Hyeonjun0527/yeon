# student-management 경로 제거 BACKLOG

## 차수 1

### 작업내용

- `/student-management` 라우트와 해당 화면 구현을 제거한다.
- 학생 운영 데모 전용 API 엔드포인트, 클라이언트 메서드, 계약 export를 함께 정리한다.
- 로그인 후 기본 이동 경로를 루트 상담 기록 랜딩으로 되돌리고 메타 문구를 현재 제품 방향에 맞춘다.

### 논의 필요

- 로그인 완료 직후 임시 진입점을 계속 `/`로 둘지, 후속 차수에서 상담 기록 워크스페이스 전용 라우트를 새로 만들지 결정이 필요하다.
- 학생 운영 데모 관련 문서와 백로그를 이번 차수에서 같이 정리할지 여부가 남아 있다.

### 선택지

- `A. /student-management 페이지만 제거`
  로그인, API, 계약 구조는 그대로 두고 화면 라우트만 없앤다.
- `B. 학생 운영 데모 축 전체 제거`
  페이지와 함께 해당 API 엔드포인트, 클라이언트 메서드, 계약 export까지 같이 정리한다.

### 추천

- `B. 학생 운영 데모 축 전체 제거`를 추천한다.
- 이유는 현재 제품 방향이 상담 기록 워크스페이스로 이동했고, 더 이상 쓰지 않는 학생 운영 계약과 엔드포인트를 남기면 로그인 후 기본 경로와 제품 의미가 다시 어긋나기 때문이다.
