# 상담 기록 워크스페이스 대규모 재설계

## 배경

현재 워크스페이스의 핵심 문제:
- 기록 없을 때 **왼쪽 사이드바에서 입력**하는 UX가 부자연스럽다. 가운데에서 해야 한다.
- 요약 카드, 핵심 태그, 원문 상태 문구 등 **의미 없는 정보**가 화면을 차지한다.
- 카드가 **너무 둥글고 공간을 비효율적으로 사용**한다. 촘촘하게 넓어져야 한다.
- 기록 리스트가 카드식이 아니라 **리스트식으로 쌓여야** 한다.
- 브랜드 컬러(오렌지 #e8630a)가 **검증되지 않은 색**이다. 전면 교체.
- **기록 삭제 기능** 없음.
- **학년/반(gradeLabel)** 필드가 불필요 → DB 스키마부터 제거.
- 가운데에 **탁 트인 AI 채팅 공간**이 있어야 한다.

## 모범 디자인 방향 (NotebookLM 참고)

```
┌─────────────┬─────────────────────────┬──────────────┐
│   왼쪽      │       가운데             │    오른쪽    │
│ 소스/기록   │    메인 채팅/원문        │  도구/출력   │
│ 리스트식    │    드래그 업로드 유도     │  AI 도우미   │
│ + 업로드    │    빈 상태: CTA 중앙     │              │
└─────────────┴─────────────────────────┴──────────────┘
```

- 왼쪽: 기록 리스트 (리스트식, 드래그 업로드 유도) + 상단에 `+ 소스 추가` 버튼
- 가운데: 선택 기록의 원문 뷰어 / 빈 상태일 때 업로드 CTA 중앙 배치
- 오른쪽: AI 채팅 도우미

## 차수 2: 의미 없는 정보 블록 전면 제거

### 작업내용

1. **요약 카드 그리드 제거** — `buildSummaryCards`, `summaryGrid`, 관련 CSS 전부 삭제
2. **핵심 태그 표시 제거** — 기록 리스트의 tags 렌더링 삭제 (`.recordTags`, `.recordTag`, `.recordTagMore`)
3. **원문 상태 문구 제거** — `STATUS_META.detail` 긴 설명 문자열 삭제, 상태 뱃지만 남김
4. **요약/액션 탭 컨텐츠 제거** — `buildSummarySections`, `buildActionItems`, 관련 뷰포트/카드/CSS 삭제
5. 탭 바 자체는 남기되, 요약/액션은 빈 상태 → 향후 실제 AI가 채울 때까지 placeholder
6. `SummaryCard`, `SummarySection`, `ActionItem` 타입 제거

### 논의 필요

- 없음. 사용자가 "전부 의미가 하나도 없어"로 결정.

### 선택지

- A. 요약/액션 탭 자체를 제거하고 원문 전체만 남김
- B. 탭은 남기되 콘텐츠를 비움

## 차수 3: 브랜드 컬러 전면 교체

### 작업내용

1. 현재 오렌지 계열(`#e8630a`) → 검증된 브랜드 컬러로 교체
2. 후보 팔레트:
   - **Slate Blue** 계열: primary `#4A6CF7`, hover `#3B5DE7` — SaaS/에듀 앱에서 검증
   - **Indigo** 계열: primary `#6366F1`, hover `#4F46E5` — Tailwind Indigo-500/600
   - **Teal** 계열: primary `#0D9488`, hover `#0F766E` — 의료/상담 도메인에서 신뢰감
3. CSS 변수 `--accent`, `--accent-hover`, `--accent-soft`, `--border-strong` 전부 교체
4. 배경 그라디언트의 오렌지 틴트 제거 → 중립 톤으로
5. 다크 모드 변수도 함께 교체

### 논의 필요

- 어떤 색 계열로 갈지. 상담/교육 도메인 = Teal or Indigo 추천.

### 선택지

- A. Indigo 계열 (`#6366F1`) — 모던, 프로페셔널
- B. Teal 계열 (`#0D9488`) — 신뢰감, 상담 도메인 적합
- C. Slate Blue 계열 (`#4A6CF7`) — SaaS 느낌

### 추천

- B. Teal. 상담 도메인에 가장 어울림.

## 차수 4: 레이아웃 구조 대전환 (NotebookLM 스타일)

### 작업내용

1. **3열 구조 재정의**:
   - 왼쪽(240~300px): 기록 리스트 + 상단 `+ 새 기록` 버튼
   - 가운데(flex-1): 원문 뷰어 OR 빈 상태 CTA OR 업로드 폼
   - 오른쪽(300~380px): AI 채팅 도우미
2. **생성 폼을 가운데로 이동**:
   - 기록 없을 때: 가운데에 드래그 업로드 영역 + CTA
   - 기록 있고 미선택일 때: "기록을 선택하세요" + 최근 기록 카드
   - `isUploadPanelOpen` 시: 가운데에 업로드 폼 표시
3. **왼쪽 사이드바 슬림화**:
   - createSection 제거 → `+ 새 기록` 버튼 하나만
   - 바로 아래 기록 리스트
4. **요약 카드/탭 제거 후** 가운데는 detailHeader + 오디오 + 원문만

### 논의 필요

## 차수 7: 공간 효율 개선 — 둥글기/패딩/갭 축소

### 작업내용

1. 전체 `border-radius` 축소:
   - 32px 패널 → 16px
   - 24px 카드 → 12px
   - 18px 요소 → 8px
   - 999px pill은 유지
2. 전체 `padding` 축소:
   - 패널 22~28px → 16~20px
   - 카드 14~18px → 10~14px
3. `gap` 축소:
   - 그리드 20px → 12~16px
   - 내부 요소 12~14px → 8~10px
4. 페이지 외곽 패딩 24px → 16px
5. 전체적으로 촘촘하고 넓게 사용하는 느낌
