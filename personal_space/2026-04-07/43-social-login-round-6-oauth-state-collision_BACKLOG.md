# 43-social-login-round-6-oauth-state-collision_BACKLOG

## 1차

### 작업내용

- 단일 OAuth state 쿠키 때문에 로그인 시도가 서로 덮어써지는 문제를 없애기 위해, pending state를 여러 개 보관하고 콜백에서 해당 state만 소비하도록 바꾼다.
- 구글과 카카오 로그인을 번갈아 시도하거나 같은 로그인 버튼을 연속으로 눌러도 먼저 시작한 인증 흐름이 불필요하게 `invalid_state`로 깨지지 않게 만든다.

### 논의 필요

- pending state를 공급자별로 분리할지, 하나의 서명 쿠키 안에서 여러 엔트리를 관리할지 여부

### 선택지

- A. 하나의 서명 쿠키 안에 여러 pending state를 넣고, 콜백에서 일치하는 항목만 제거한다
- B. 공급자마다 별도 쿠키를 두고 provider 단위로만 충돌을 줄인다

### 추천

- A. 같은 공급자 중복 시도와 다른 공급자 교차 시도를 모두 막을 수 있어 실제 인증 안정성이 더 높다.
