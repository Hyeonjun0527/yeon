# 상담 기록 워크스페이스 선택 오디오 미리보기 정리

## 목적

- [66-counseling-workspace-primary-cta-block.md](/home/osuma/coding_stuffs/yeon/personal_space/2026-04-07/66-counseling-workspace-primary-cta-block.md)에서 정의한 `선택한 오디오 확인 -> 기록 저장` 흐름 안에서, 오디오 카드가 너무 무겁지 않게 역할을 다시 고정한다.
- 이번 차수의 핵심은 `오디오 정보를 더 넣는 것`이 아니라, `저장 전에 필요한 확인만 남기는 것`이다.
- 구현 단계에서는 이 문서를 기준으로 선택 오디오 카드의 정보 구성, 상태 표현, 플레이어 노출 방식을 정리한다.

## 현재 문제

## 코드 기준

- 현재 선택 오디오 카드는 `파일명 + 크기/길이 + 업로드 준비 pill + native audio controls` 구조다.
  - `counseling-record-workspace.tsx:1663-1682`
- 파일 선택 직후에는 카드가 이미 있는데도 별도로 `파일명 선택 완료` 성공 메시지가 한 번 더 나타난다.
  - `counseling-record-workspace.tsx:1073-1078`
  - `counseling-record-workspace.tsx:1706-1717`
- 카드 스타일도 현재는 generic panel 계열과 같은 감각이라, 확인 블록치고 약간 두껍게 보일 수 있다.
  - `counseling-record-workspace.module.css:402-440`
- `업로드 준비` pill은 의미가 나쁘진 않지만, 저장 전 상태를 이미 카드 자체가 말하고 있어 정보가 중복된다.
  - `counseling-record-workspace.tsx:1675`

## UX 판단

- 사용자가 이 단계에서 알고 싶은 건 딱 3가지다.
- `내가 고른 오디오가 맞는지`
- `대략 어느 정도 길이/크기인지`
- `바로 저장해도 되는지`
- 지금 구조는 크게 잘못되진 않았지만, 같은 의미를 `성공 메시지 + pill + 카드`로 반복해서 말한다.

## 최종 결정

## 선택

- 백로그 추천안 `A. 재생기를 바로 노출`로 고정한다.

## 이유

- 오디오 선택 직후 가장 자연스러운 다음 행동은 `짧게 들어보고 맞는지 확인`하는 것이다.
- `미리 듣기` 버튼을 한 번 더 두면 저장 전 확인 흐름이 오히려 늘어진다.
- native audio controls는 이미 충분히 익숙하고, 별도 커스텀 플레이어를 만들 이유도 없다.

## 오디오 카드 역할 재정의

### 지금까지

- `선택 성공 상태를 포함한 정보 카드`

### 앞으로

- `저장 전 오디오 확인 블록`

## 반드시 남길 정보

- 파일명
- 크기와 길이
- 간단한 준비 상태
- native 오디오 플레이어

## 제거하거나 약하게 만들 정보

- 별도 성공 메시지 중복
- 강한 상태 pill
- 긴 설명문
- 카드 안의 추가 카드

## 최종 구조

```text
선택한 오디오
파일명
24.8MB · 18분 12초 · 저장 준비

[ native audio controls ]
```

## 구조 규칙

- 카드 상단에 `선택한 오디오` 같은 작은 맥락 라벨은 허용한다.
- 파일명이 가장 먼저 읽혀야 한다.
- 크기/길이/상태는 한 줄 메타로 묶는다.
- 플레이어는 바로 아래에 바로 노출한다.
- `다른 파일 선택`, `브라우저 녹음` 같은 보조 행동은 카드 안이 아니라 카드 바깥 support action row로 둔다.

## 준비 상태 표현

## 유지 여부

- 준비 상태는 유지하되, 별도 pill 대신 메타 줄 안으로 흡수하는 방향을 추천한다.

## 권장 표현

- `24.8MB · 18분 12초 · 저장 준비`

## 이유

- 지금 단계의 상태는 `ready` 하나뿐이라 강한 badge가 필요하지 않다.
- 이미 저장 버튼이 곧 아래에 있으므로, 상태는 보조 텍스트 수준이면 충분하다.
- `readyPill`과 `파일 선택 완료` 메시지를 동시에 두면 시선만 갈라진다.

## 성공/오류 메시지 규칙

## 성공 메시지

- 파일 선택 성공을 별도 인라인 메시지로 다시 보여주지 않는다.
- 카드가 렌더된 순간 자체가 성공 피드백 역할을 한다.

## 유지할 메시지

- 파일 크기 초과
- 오디오 형식 오류
- 녹음 권한 오류
- 저장 실패

## 의미

- `선택 완료` 같은 정상 메시지는 카드 구조로 대체
- 실제 사용자 대응이 필요한 예외 메시지만 인라인으로 남김

## 플레이어 노출 규칙

## 기본값

- 항상 바로 노출

## 위치

- 메타 줄 아래
- 카드의 마지막 요소

## 스타일

- 커스텀 컨트롤 금지
- 전체 폭 사용
- 카드 안에서만 한 번 보인다

## 예외

- `selectedAudioPreviewUrl`이 없을 때는 플레이어를 숨기되, 별도 장문 설명을 넣지 않는다.
- 이 경우 `미리듣기를 준비하지 못했습니다.` 수준의 짧은 한 줄만 허용 가능하다.

## 시각 규칙

## 카드 톤

- `selectedAudioCard`는 강한 카드가 아니라 약한 확인 블록처럼 보여야 한다.
- 배경은 `surface-soft` 수준
- 보더는 약하게 유지
- 그림자 없음
- 반경은 16px 안팎

## 타이포

- 파일명: 15-16px, 600-700
- 메타: 12-13px, muted
- 별도 제목 라벨을 쓰면 12px 이하 보조 톤

## 간격

- 헤더와 플레이어 간 10-12px
- 카드 내부 padding 14-16px
- 저장 버튼과 카드 사이 간격은 12-16px

## 금지

- 업로드 성공 축하 문구
- 두꺼운 accent border
- `업로드 준비` badge와 성공 메시지의 동시 노출
- 파일명 아래 긴 설명문

## 모바일 규칙

- 모바일에서는 현재처럼 헤더가 세로로 떨어져도 괜찮다.
- 다만 pill 제거 후에는 제목/메타/플레이어의 세로 흐름이 더 자연스러워야 한다.
- 플레이어는 full width 유지

## 카피 개선안

| 현재 | 변경 |
| --- | --- |
| `업로드 준비` | `저장 준비`를 메타 줄에 흡수 |
| `example.m4a 선택 완료` | 삭제 |
| 없음 | `선택한 오디오` 보조 라벨 추가 가능 |

## 구현 메모

- `readyPill`은 이 카드에서는 제거하거나 사용 범위를 다른 상태 칩으로 한정하는 편이 낫다.
- `applySelectedAudioFile`에서 설정하는 `선택 완료` 메시지는 tone이 success일 때 카드 렌더로 대체하는 분기 정리가 필요하다.
- 파일 교체 액션이 필요하면 카드 내부가 아니라 카드 아래 보조 액션 행으로 배치한다.

## 이번 차수의 완료 판단

- 선택 오디오 카드가 `파일명 -> 메타 -> 플레이어` 순서로 간결하게 읽히는지
- `선택 완료` 메시지와 `업로드 준비` badge 중복이 제거됐는지
- 플레이어가 별도 클릭 없이 바로 보여지는지
- 카드가 설명 박스가 아니라 저장 전 확인 블록처럼 느껴지는지
