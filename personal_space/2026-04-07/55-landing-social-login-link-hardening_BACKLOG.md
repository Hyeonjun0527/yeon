## 1차

### 작업내용

- 랜딩 로그인 모달의 소셜 로그인 진입이 JS 이벤트에 과도하게 의존하는 부분을 점검한다.
- 소셜 로그인 버튼을 실제 링크 기반으로 바꿔, 랜딩의 서드파티 런타임 오류가 있어도 이동이 되게 만든다.
- 로컬 브라우저에서 카카오 로그인 버튼 클릭 시 `/api/auth/kakao` 경유 후 외부 로그인 페이지로 넘어가는지 검증한다.

### 논의 필요

- 랜딩 1차 CTA를 계속 모달 오픈으로 둘지, 바로 소셜 선택 화면으로 보낼지

### 선택지

- 선택지 1: 현재처럼 JS `window.location.assign` 유지
- 선택지 2: 소셜 로그인 버튼을 `href` 기반 링크로 변경
