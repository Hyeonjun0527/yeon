# 상담 기록 워크스페이스 30차 디자인 개선안 계획

## 기준

- 대상: `apps/web/src/features/counseling-record-workspace/`
- 이전 차수: 6~13차 (헤더 슬림, 빈 상태 셸, CTA 블록, 점진적 폼, 디스클로저, 오디오 미리보기, 녹음 마이크로플로우, 사이드바 IA)
- 이번 계획: 14~43차
- 각 차수는 설계 문서 → 구현 → 검증 순서로 진행

---

## Phase 1: 좌측 탐색 마무리 (14~16차)

### 14차: 검색/필터 노출 강도 조정

- 검색창 높이를 48px → 44px로 줄이고, 필터 칩을 기본 접힘 토글 뒤로 숨긴다.
- 필터가 `all`이 아닐 때만 활성 필터 1개를 인라인 텍스트로 보여준다.
- 기록 5건 이하일 때는 검색/필터 행 자체를 숨긴다.
- 목표: 리스트가 도구보다 먼저 읽히게 만든다.

### 15차: 기록 리스트 아이템 밀도 축소

- 현재 recordItem은 `이름 + 학년 + 제목 + 날짜/길이 + preview + tags`가 모두 보인다.
- preview를 1줄 말줄임으로 줄이고, tags는 2개까지만 + `+n` 처리한다.
- 선택 상태의 좌측 accent bar를 추가해 선택/비선택을 색만이 아닌 구조로 구분한다.
- recordItem 패딩을 18px → 14px로 줄여 같은 높이에 더 많은 기록을 노출한다.
- 목표: 스크롤 없이 4~5개 기록이 보이게 만든다.

### 16차: 좌측 스크롤/고정 구조 정리

- createSection은 스크롤하지 않고 상단 고정, browseSection만 내부 스크롤한다.
- sidebar를 `grid-template-rows: auto minmax(0, 1fr)`로 재구성한다.
- recordList에 `overscroll-behavior: contain`을 적용해 페이지 바운스를 막는다.
- sticky header가 리스트 위에 그림자를 줄지 않게 처리한다.
- 목표: 생성 CTA는 항상 접근 가능하고, 긴 목록은 자연스럽게 스크롤된다.

## Phase 2: 중앙 상세 영역 정리 (17~22차)

### 17차: 상세 헤더 정보 위계 재정의

- 현재 detailHeader에 `제목 + 설명 + metaChips 7개 + audioPanel + retryState + actionRow`가 밀집돼 있다.
- metaChips를 3개 핵심(날짜, 길이, 상담유형)만 남기고 나머지는 접힌 `상세 정보` 행으로 내린다.
- 설명문을 sessionTitle 한 줄로 이미 줄였으므로, detailFootnote도 제거한다.
- 목표: 상세 헤더가 정보 보드가 아니라 기록 식별 스트립으로 읽히게 만든다.

### 18차: 오디오 플레이어 영역 슬림화

- audioPanel의 eyebrow + 제목 + meta 3개 + 플레이어 + 힌트 문장 구조를 정리한다.
- 제목을 제거하고, 파일명과 크기를 플레이어 바로 위 1줄 메타로 흡수한다.
- 현재 재생 시간 chip은 플레이어 자체가 보여주므로 제거한다.
- 힌트 문장 `원문 구간을 클릭하면...`은 최초 1회만 보여주거나 제거한다.
- 목표: 플레이어가 주인공이고, 부가 텍스트가 최소화된 구조.

### 19차: 새로고침/재전사 액션 행 축소

- 현재 `최신 상태 새로고침`과 `전사 다시 시도`가 항상 보인다.
- 새로고침은 status가 processing일 때만 자동으로 돌아가므로, ready 상태에서는 숨기거나 약화한다.
- 재전사는 error 상태에서만 primary로 올리고, 그 외에는 overflow 메뉴 뒤로 숨긴다.
- retryState 메시지는 toast나 임시 인라인으로 바꿔 영구 노출을 줄인다.
- 목표: 상태에 맞는 액션만 보이게 만든다.

### 20차: 요약 카드 그리드 재구성

- 현재 summaryGrid는 4열(1개 wide + 3개 normal)인데 콘텐츠가 템플릿 기반이라 정보량이 적다.
- 4개 카드를 2개로 줄이고, 나머지는 요약 탭 안으로 이동한다.
- 카드 높이를 고정하지 않고 콘텐츠에 맞게 자연스럽게 흐르게 한다.
- 카드 스타일을 더 가볍게 만들어 뷰어 패널과 시각적 경쟁을 줄인다.
- 목표: 요약 카드가 헤더와 뷰어 사이의 부담이 아닌 짧은 맥락 제공자로 읽히게 만든다.

### 21차: 탭 바 + 뷰어 헤더 병합

- 현재 tabBar와 viewerHeader가 별도 행으로 있어 뷰어 진입 전 높이가 두꺼워진다.
- tabBar와 viewerTitle을 같은 행에 두고, 검색은 탭 우측에 인라인으로 배치한다.
- viewerMeta chips를 제거하거나 탭 옆에 1개만 남긴다.
- 목표: 탭 선택과 콘텐츠 시작 사이의 간격을 최소화한다.

### 22차: 원문 세그먼트 행 타이포/간격 미세 조정

- segmentText를 16px에서 15px로 줄이고 line-height를 1.9에서 1.75로 조정한다.
- segmentTime과 segmentSpeaker 열의 고정 폭을 줄여 텍스트 열에 더 많은 공간을 준다.
- 세그먼트 간 구분선을 더 약하게 만들어 텍스트 흐름이 끊기지 않게 한다.
- active segment의 시각적 강조를 좌측 bar + 약한 배경으로 통일한다.
- 목표: 긴 원문을 읽을 때 시각적 피로가 줄어든다.

## Phase 3: 우측 AI 패널 재설계 (23~26차)

### 23차: AI 패널 헤더/문맥 영역 슬림화

- assistantHeader의 아이콘 + eyebrow + 제목 + 설명을 줄인다.
- 아이콘을 작게 하거나 제거하고, 제목만 남긴다.
- assistantContext의 문맥 고정 카드 안의 contextMeta chips + statusParagraph를 1줄 요약으로 줄인다.
- 목표: AI 패널의 고정 영역을 줄여 대화 영역에 더 많은 공간을 확보한다.

### 24차: 빠른 프롬프트 칩 재배치

- 현재 promptChip 3개가 패널 중간에 고정돼 있어 대화가 길어지면 밀어올려진다.
- 프롬프트 칩을 composer 위로 이동하거나, 대화가 시작되면 접히게 만든다.
- 첫 대화 전에는 칩이 중앙에 크게 보이고, 대화 시작 후에는 composer 위 작은 행으로 남긴다.
- 목표: 프롬프트 칩이 대화 흐름을 방해하지 않으면서도 접근 가능하게 유지된다.

### 25차: 메시지 버블 스타일 개선

- 현재 assistant/user 메시지가 같은 카드 스타일인데, 역할 구분이 배경색에만 의존한다.
- assistant 메시지는 좌측 정렬, user 메시지는 우측 정렬로 채팅 방향을 구분한다.
- messageBadge를 제거하고 정렬 방향 자체가 역할을 말하게 만든다.
- supportingNote를 작은 캡션으로 더 약하게 만든다.
- 목표: 대화 흐름이 한눈에 읽히는 채팅 UI로 전환한다.

### 26차: Composer 입력 영역 개선

- textarea rows를 4에서 2로 줄이고, 입력량에 따라 자동 확장한다.
- 전송 버튼을 textarea 우측 하단 인라인으로 이동해 공간을 절약한다.
- `Enter`로 전송, `Shift+Enter`로 줄바꿈 규칙을 적용한다.
- 목표: composer가 화면의 많은 비중을 차지하지 않으면서 빠르게 입력할 수 있다.

## Phase 4: 전체 레이아웃/반응형 (27~30차)

### 27차: 선택 전 빈 상태 셸 시각 보강

- 현재 preSelectionShell이 텍스트만 있어 너무 비어 보인다.
- 간단한 일러스트나 아이콘을 추가하되 과하지 않게 한다.
- 기록이 있을 때는 최근 생성/수정된 기록 1~2개를 빠른 선택 카드로 보여준다.
- 목표: 빈 상태가 미완성이 아니라 다음 행동을 안내하는 구조로 읽힌다.

### 28차: 1360px 2열 레이아웃 개선

- 현재 1360px 이하에서 assistantPanel이 centerColumn 아래로 내려가면서 컨텍스트가 끊긴다.
- 대안: assistantPanel을 우측 슬라이드 드로어로 전환하거나, 중앙 하단 접힌 패널로 변경한다.
- 2열 모드에서 sidebar와 center의 비율을 다시 조정한다.
- 목표: 태블릿 화면에서 AI 패널이 자연스럽게 접근 가능하다.

### 29차: 1080px 이하 모바일 레이아웃 전면 재설계

- 모바일에서 sidebar → center → assistant 순서를 유지하되, 각 영역을 탭 또는 바텀시트로 전환한다.
- 원문 세그먼트를 1열에서도 읽기 좋게 시간/화자를 텍스트 위 인라인으로 재배치한다.
- 생성 CTA를 FAB(Floating Action Button)로 올린다.
- 목표: 모바일에서도 핵심 워크플로우가 끊기지 않는다.

### 30차: 720px 이하 초소형 화면 대응

- 720px 이하에서 detailHeader의 metaChips를 완전히 숨기고 제목만 남긴다.
- 탭 바를 하단 고정 내비게이션으로 전환한다.
- 오디오 플레이어를 하단 미니 바로 전환해 스크롤 중에도 접근 가능하게 한다.
- 목표: 스마트폰에서 최소한의 원문 검토가 가능하다.

## Phase 5: 인터랙션/피드백 (31~34차)

### 31차: 원문 재생 시 자동 스크롤

- 오디오 재생 중 active segment가 viewport에 없으면 자동으로 스크롤한다.
- smooth scroll + 사용자가 수동 스크롤 중이면 자동 스크롤을 일시 중지한다.
- `자동 스크롤 따라가기` 토글을 뷰어 상단에 둔다.
- 목표: 30분 이상 긴 음성에서도 재생과 원문이 동기화된다.

### 32차: 기록 저장 성공 피드백 강화

- 현재 저장 성공 후 inline message만 뜨는데, 새 기록이 바로 선택되면서 메시지가 묻힌다.
- 저장 직후 짧은 toast 알림 + 사이드바에서 새 기록 카드에 잠깐 highlight 효과를 준다.
- 목표: 사용자가 저장이 완료됐고 어떤 기록이 생겼는지를 확실히 인지한다.

### 33차: 로딩 상태 스켈레톤 도입

- 현재 로딩 중에는 텍스트 빈 상태만 보인다.
- 기록 리스트, 상세 헤더, 원문 세그먼트에 스켈레톤 placeholder를 도입한다.
- 스켈레톤은 실제 콘텐츠와 비슷한 형태로 만들어 레이아웃 시프트를 줄인다.
- 목표: 로딩 중에도 화면이 완성된 것처럼 보인다.

### 34차: 에러 상태 가이드 개선

- 전사 실패 시 `errorMessage`만 보여주는 대신, 원인별 분기 안내를 추가한다.
- `파일이 손상됐을 때`, `서버 시간초과일 때`, `형식 미지원일 때` 각각 다른 안내.
- 재전사 버튼 옆에 `다른 파일로 교체` 옵션을 제공한다.
- 목표: 에러 상태에서도 사용자가 길을 잃지 않는다.

## Phase 6: 시각 정체성/톤 (35~38차)

### 35차: 색상 팔레트 정리

- 현재 accent(#e8630a) 하나로 CTA, 필터, 검색 강조, 상태를 모두 처리한다.
- accent를 행동 전용으로 좁히고, 정보 강조에는 neutral tone을 쓴다.
- danger/success 외에 warning(processing 전용) 톤을 분리한다.
- 목표: 색이 역할을 명확히 구분한다.

### 36차: 타이포그래피 스케일 정리

- 현재 11~38px 사이에 스케일이 비정형적으로 흩어져 있다.
- 5~6단계 타이포 스케일(12, 14, 16, 20, 28, 36)로 고정하고, 예외를 줄인다.
- font-weight도 400/600/700 3단계로 정리한다.
- 목표: 글자 크기만으로 위계가 읽히는 일관된 스케일.

### 37차: 그림자/보더/반경 일관성 정리

- 현재 border-radius가 10, 12, 14, 16, 18, 22, 24, 32, 999px로 흩어져 있다.
- 4단계(8, 16, 24, 999)로 줄이고 용도별로 고정한다.
- 그림자도 2단계(md, lg)를 명확히 구분하고, 불필요한 box-shadow를 제거한다.
- 목표: 시각적 노이즈가 줄고 표면이 정돈된 느낌.

### 38차: 전환/모션 일관성 정리

- 현재 transition이 0.18s ease로 거의 통일돼 있지만, hover와 상태 전환이 모두 같은 속도다.
- hover는 120ms, 상태 전환(패널 열기/닫기)은 200ms, 강조(active segment)는 300ms로 분리한다.
- `prefers-reduced-motion`에서 모든 전환을 제거한다.
- 목표: 빠른 인터랙션과 부드러운 전환이 구분된다.

## Phase 7: 고급 기능 UX (39~43차)

### 39차: 원문 내 키워드 하이라이트 강화

- 현재 검색 일치만 강조하는데, 사용자가 특정 단어를 영구 하이라이트할 수 있게 한다.
- 원문에서 텍스트를 드래그해 `하이라이트` 버튼으로 마킹하는 인터랙션.
- 하이라이트된 단어는 요약 탭에서 빈도와 함께 정리된다.
- 목표: 원문 검토 과정에서 핵심 키워드를 능동적으로 정리할 수 있다.

### 40차: 세그먼트별 메모/주석 UX

- 특정 세그먼트에 마우스를 올리면 `메모 추가` 버튼이 나타난다.
- 메모는 세그먼트 아래 인라인으로 접힌 채 표시되고, 아이콘으로 존재를 알린다.
- 메모가 있는 세그먼트는 좌측에 작은 마커를 표시한다.
- 목표: 원문 검토 중 발견한 포인트를 바로 기록할 수 있다.

### 41차: 원문/요약 내보내기 UI

- 상세 헤더 또는 뷰어 상단에 `내보내기` 버튼을 둔다.
- 원문 전체, 요약만, 액션 아이템만, 또는 하이라이트된 부분만 선택 내보내기.
- 형식은 텍스트/마크다운/클립보드 복사 중 선택.
- 목표: 검토 결과를 외부(보호자 공유, 팀 문서)에 바로 활용할 수 있다.

### 42차: 키보드 내비게이션 체계

- `↑/↓`로 기록 리스트 탐색, `Enter`로 선택.
- `Tab`으로 좌측 → 중앙 → 우측 패널 이동.
- `Ctrl+K`로 글로벌 검색 포커스.
- `Space`로 오디오 재생/일시정지.
- `1/2/3`으로 탭 전환.
- 포커스 링을 모든 인터랙티브 요소에 일관되게 적용한다.
- 목표: 마우스 없이도 전체 워크플로우가 가능하다.

### 43차: 다크 모드 지원

- CSS 변수 기반으로 다크 팔레트를 추가한다.
- `prefers-color-scheme: dark` 미디어 쿼리로 자동 전환.
- 그라데이션, 그림자, accent 색상을 다크 모드에 맞게 조정한다.
- 이미지/아이콘 색상도 다크 모드에서 반전하지 않고 자연스럽게 매칭한다.
- 목표: 야간/어두운 환경에서도 시각적 피로 없이 작업 가능하다.
