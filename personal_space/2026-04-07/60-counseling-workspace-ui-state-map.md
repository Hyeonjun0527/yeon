# 상담 기록 워크스페이스 상태별 화면 맵

## 목적

- [59-counseling-workspace-ui-diagnosis.md](/home/osuma/coding_stuffs/yeon/personal_space/2026-04-07/59-counseling-workspace-ui-diagnosis.md)의 진단 결과를 실제 화면 상태로 번역한다.
- 이후 구현은 이 문서의 상태 맵을 source of truth로 삼는다.
- 기준은 `처음에는 적게 보이고`, `행동 이후에만 풍부해진다`, `AI는 선택 후에만 등장한다`이다.

## 고정 원칙

### 1. 첫 화면의 질문은 하나다

- 사용자가 첫 진입에서 답해야 하는 질문은 `업로드할까, 녹음할까` 하나뿐이다.
- 학생 정보, 상담 유형, AI 보조, 원문 탭, 상태 수치가 먼저 나오면 안 된다.

### 2. 빈 상태는 패널 1개 감각으로 보여야 한다

- 데이터가 없거나 기록이 선택되지 않았을 때는 `여러 개의 빈 박스`가 아니라 `하나의 분명한 다음 행동`이 보여야 한다.
- 우측 AI 패널은 선택 전 레이아웃에서 제거한다.

### 3. 입력은 점진적으로 노출한다

- 첫 노출: `파일 업로드`, `브라우저 녹음`
- 두 번째 노출: 선택한 오디오 확인, 저장
- 세 번째 노출: 학생명, 상담 유형 같은 상세 입력

### 4. 상태 정보는 하위로 내린다

- 기록 수, 준비 완료 수, 세그먼트 수, 모델명, 필터 카운트는 보조 정보다.
- 상단 주인공 자리에 두지 않는다.

### 5. 원문과 AI의 등장 순서를 지킨다

- 선택 전: AI 없음
- 선택 직후: 원문과 핵심 요약 먼저
- 이후: AI 패널 또는 AI 토글 노출

## 상태 전이 개요

```text
첫 진입
  -> 업로드/녹음 시작
생성 흐름
  -> 기록 저장 완료
목록 탐색
  -> 기록 선택
기록 선택 후
  -> 선택 해제 또는 다른 기록 선택
목록 탐색
```

## 상태 1. 첫 진입

### 진입 조건

- 기록이 0건이거나
- 기록이 있어도 아직 아무 기록도 선택하지 않았고
- 생성 패널도 열리지 않은 기본 진입 상태

### 사용자가 바로 이해해야 하는 것

- `여기서 파일을 올리거나 바로 녹음하면 기록이 생긴다`

### 레이아웃

| 영역 | 노출 요소 | 노출 이유 |
| --- | --- | --- |
| 상단 헤더 | 페이지 제목, 한 줄 보조 문구, 약한 보조 액션 | 제품 설명보다 작업 진입을 먼저 보여준다. |
| 좌측 | 대형 CTA 2개 `파일 업로드`, `브라우저 녹음` | 첫 행동의 source of truth |
| 중앙 | 짧은 빈 상태 한 덩어리 `기록을 만들면 원문과 요약이 여기 열립니다` | 여러 박스 대신 단일 방향 제시 |
| 우측 | 없음 | 선택 전 AI는 의미가 없다. |

### 반드시 숨길 것

- 상단 메트릭 카드
- 학생명/상담유형 입력
- 검색창/필터칩
- 중앙 대형 빈 패널 반복
- 우측 AI 빈 상태 패널

### 보조 문구 허용 범위

- 1줄만 허용
- 예: `원문 확인, 요약 정리, 후속 조치까지 이어집니다.`

### 전이 트리거

- `파일 업로드` 클릭
- `브라우저 녹음` 클릭
- 기존 기록이 있다면 `기록 열기` 클릭

## 상태 2. 생성 흐름

### 진입 조건

- 업로드 패널이 열렸거나
- 녹음이 시작됐거나
- 파일이 선택됐지만 아직 저장 전인 상태

### 사용자가 바로 이해해야 하는 것

- `오디오를 확인하고 저장하면 기록이 생성된다`

### 레이아웃

| 영역 | 노출 요소 | 노출 이유 |
| --- | --- | --- |
| 상단 헤더 | 상태 1과 동일 | 생성 중에도 상단은 조용해야 한다. |
| 좌측 상단 | CTA 2개 유지 + 선택된 오디오 카드 + 저장 버튼 | 생성 흐름의 핵심 동선 |
| 좌측 하단 | `추가 정보` 접기 섹션 | 상세 입력은 선택사항으로 밀어낸다. |
| 중앙 | 짧은 진행 안내 또는 최근 기록 1~2개 | 생성 과정에서 화면이 비어 보이지 않게 하되 주인공이 되지 않게 한다. |
| 우측 | 없음 | 선택 전 AI는 여전히 숨긴다. |

### 세부 서브상태

| 서브상태 | 보여줄 것 | 숨길 것 |
| --- | --- | --- |
| 업로드 전 | CTA 2개, 짧은 안내 | 학생 정보 폼 전체 |
| 파일 선택 후 | 파일명, 길이, 미리듣기, 저장 버튼 | 긴 업로드 설명 |
| 녹음 중 | 녹음 타이머, 중지 액션, 저장 대기 상태 | 다른 보조 필드 |
| 저장 중 | 진행 문구 1줄, 버튼 loading | 추가 입력, 우측 패널 |

### 반드시 숨길 것

- 상태 필터칩
- AI 패널
- 상세 헤더/원문 탭
- 요약 카드

### 추천 인터랙션

- `추가 정보`는 기본 닫힘
- 저장 후에는 성공 토스트보다 `방금 생성한 기록`을 목록 상단에 즉시 반영하는 편이 중요하다

### 전이 트리거

- 저장 성공 -> `목록 탐색`
- 취소/닫기 -> `첫 진입` 또는 기존 기록이 있으면 `목록 탐색`

## 상태 3. 목록 탐색

### 진입 조건

- 기록이 1건 이상 존재하고
- 아직 어떤 기록도 열지 않은 상태
- 또는 생성 직후 기록이 생겼지만 본문 상세는 아직 열지 않은 상태

### 사용자가 바로 이해해야 하는 것

- `방금 만든 기록이나 기존 기록 중 하나를 열면 된다`

### 레이아웃

| 영역 | 노출 요소 | 노출 이유 |
| --- | --- | --- |
| 상단 헤더 | 제목, 한 줄 보조 문구 | 상단은 여전히 조용하게 유지 |
| 좌측 상단 | `새 기록 만들기` 버튼 또는 접힌 CTA | 첫 행동으로 다시 돌아갈 수 있어야 한다. |
| 좌측 본문 | 기록 리스트 | 핵심 탐색 행동 |
| 좌측 보조 | 검색창은 약하게 노출, 필터는 토글 안 | 기록이 늘어날수록 탐색이 필요하다. |
| 중앙 | `기록을 선택하세요` 안내 + 최근/방금 생성한 기록 빠른 액션 | 빈 박스가 아니라 다음 행동 안내 |
| 우측 | 없음 | 선택 전 AI 제거 유지 |

### 검색/필터 노출 규칙

- 검색창은 기록 수가 적어도 있어도 되지만 시각적 강조를 낮춘다.
- 필터칩은 항상 보이지 않고 `필터` 버튼 뒤에 배치한다.
- `전체 n건`, `준비 완료 n건` 같은 수치 요약은 상단이 아니라 목록 보조 텍스트로만 약하게 둔다.

### 생성 직후 추가 규칙

- 방금 생성된 기록은 목록 맨 위에 온다.
- 방금 생성된 기록은 `새 기록` 정도의 약한 표시만 허용한다.
- 아직 원문 준비 중이면 중앙 안내는 `기록은 생성됨, 원문 준비 중` 정도로 짧게 둔다.

### 반드시 숨길 것

- 우측 AI 패널 전체
- 상세 헤더 전체
- 원문/요약/액션 탭
- 상세 메타칩

### 전이 트리거

- 기록 행 클릭 -> `기록 선택 후`
- `새 기록 만들기` -> `생성 흐름`

## 상태 4. 기록 선택 후

### 진입 조건

- 사용자가 하나의 기록을 선택한 상태

### 사용자가 바로 이해해야 하는 것

- `이제 원문을 읽고, 필요한 요약과 AI 보조를 사용하면 된다`

### 레이아웃

| 영역 | 노출 요소 | 노출 이유 |
| --- | --- | --- |
| 상단 헤더 | 제목, 현재 작업 컨텍스트 1줄, 약한 보조 액션 | 헤더는 여전히 얇게 유지 |
| 좌측 | 접힌 `새 기록 만들기` + 기록 리스트 | 현재 작업 중에도 다른 기록 전환이 쉬워야 한다. |
| 중앙 상단 | 선택 기록 제목, 핵심 상태, 최소 메타 | 문맥 확인 |
| 중앙 본문 | 원문 탭 기본, 짧은 요약 블록, 액션 탭 | 원문이 주인공 |
| 우측 | AI 보조 패널 또는 AI 토글 | 이 단계에서만 보조 기능이 의미를 가진다. |

### 중앙 영역 우선순위

1. 원문 전체
2. 짧은 구조화 요약
3. 후속 액션
4. 상세 메타

### 우측 AI 패널 우선순위

1. 패널 제목
2. 퀵프롬프트 일부
3. 대화 영역
4. 입력창
5. 접힌 컨텍스트 정보

### 반드시 줄일 것

- 상세 헤더 긴 설명문
- 메타칩 과다 노출
- 요약 카드 4장 중첩 구조
- AI 패널 상단 설명문과 문맥 배지 나열

### 선택 후에만 허용되는 요소

- 원본 음성 패널
- 재전사 액션
- 원문 검색
- AI 질문 입력

### 전이 트리거

- 다른 기록 선택 -> 같은 상태 유지, 컨텍스트 교체
- 선택 해제 -> `목록 탐색`
- 새 기록 시작 -> `생성 흐름`

## 구현 시 바로 적용할 결정

- 기본 진입에서는 3열 구조를 시각적으로 완성하지 않는다.
- 우측 AI 패널은 `selectedRecord`가 있을 때만 마운트하는 것이 기본값이다.
- 생성 흐름은 `CTA -> 오디오 확인 -> 저장`이 먼저고, 상세 입력은 뒤다.
- 기록이 존재하지만 선택 전이면 중앙은 `선택 유도 상태`여야지, 거대한 빈 박스가 아니어야 한다.
- 선택 후 화면에서 가장 넓은 영역은 항상 원문이다.

## 차수 3 입력값

- 차수 3에서는 이 상태 맵을 기준으로 각 요소를 `1차 정보`, `2차 정보`, `보조 정보`, `숨김 후보`로 더 세밀하게 분류한다.
