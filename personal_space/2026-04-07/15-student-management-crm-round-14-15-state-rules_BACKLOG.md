# 15-student-management-crm-round-14-15-state-rules_BACKLOG

## 현재 상태

- 홈 화면은 공모전 설명과 학생관리 프리뷰가 섞여 있어 실제 학생관리 도구 첫 화면으로는 집중도가 낮다.
- round 14에서 요구한 375px 기준 모바일 밀도 조정은 일부 반응형 규칙만 있고, 학생관리 핵심 행동을 중심으로 다시 정리된 제품 홈은 아직 없다.
- round 15에서 요구한 로딩, 비어 있음, API 실패, 데이터 누락 상태 화면 규칙도 현재 홈에는 명시적으로 없다.
- 현재 워크트리에는 다른 에이전트의 미커밋 변경이 있어, 기존 `contest-home` 대규모 수정은 충돌 위험이 높다.

## 이번 진행 차수

- `14차 + 15차 통합`

## 목표 상태

- 기본 홈이 설명형 랜딩보다 실제 학생관리 업무 화면에 가깝게 재구성된다.
- 375px 모바일에서도 핵심 브리핑, 우선 학생 큐, 상세 신호가 한 열 흐름으로 안정적으로 읽힌다.
- 로딩, 비어 있음, API 실패, 데이터 누락 상태가 같은 디자인 시스템과 CTA 규칙으로 정의된다.
- 기존 미커밋 파일을 덮지 않도록 새 feature를 추가하고 엔트리만 최소 교체한다.

## 14차

- 기존 공모전 설명 비중을 줄이고 오늘 행동 중심의 제품 구조로 재배치한다.

- A. 학생관리 화면 중심으로 재구성하고 공모전 문맥은 보조 설명으로만 남긴다
- B. 기존 설명형 섹션을 그대로 유지한다

### 추천
