# 78. 학생별 상담 이력 뷰 + AI 추이 분석

## 예상 차수: 5차

## 현황

- 상담 기록은 시간순 flat list로만 표시
- 같은 학생의 기록을 모아보려면 사이드바 검색으로 학생명을 입력해야 함
- AI 분석은 단일 기록 단위. 학생의 변화 추이를 기록 간 연결해서 분석하는 기능 없음
- DB에 `studentName`은 varchar — 학생 엔티티가 분리되어 있지 않음

## 목표

- 학생 이름 기준으로 상담 기록을 그룹화해서 이력 타임라인으로 보여줌
- AI가 여러 기록을 참조해 학생의 변화 추이를 분석
- 장기적으로 학생 엔티티 분리의 기반 마련

---

### 차수 1: 서버 — 학생별 기록 조회 API

**작업내용**

- `GET /api/v1/counseling-records?groupBy=student` — 학생별 그룹 응답
- 또는 `GET /api/v1/students/summary` — 학생 이름별 기록 수, 최근 상담일, 첫 상담일
- 현재 `studentName` varchar 기반 그룹핑 (엔티티 분리 전)
- api-contract에 응답 스키마 추가

**논의 필요**

- 학생 엔티티를 지금 분리할지, 이름 기반으로 먼저 가고 나중에 분리할지

**선택지**

- A) `studentName` 기반 GROUP BY — 빠르게 구현, 동명이인 이슈
- B) `students` 테이블 생성, FK 연결 — 정석이지만 마이그레이션 + 기존 데이터 정리 필요

**추천**: A) 이름 기반으로 먼저 출시. 사용자가 실제로 쓰기 시작한 뒤 동명이인 문제가 발생하면 그때 엔티티 분리.

### 차수 2: 클라이언트 — 사이드바 학생 그룹 뷰

**작업내용**

- 사이드바에 "전체 / 학생별" 뷰 전환 토글
- 학생별 뷰: 학생 이름 → 해당 학생 기록 목록 (아코디언 또는 들여쓰기)
- 각 학생 그룹 헤더: 이름, 기록 수, 최근 상담일
- 학생 그룹 클릭 → 해당 학생 기록만 필터링

**논의 필요**: 없음

**추천**: 아코디언 형태. 기존 리스트 UI를 최대한 재활용.

### 차수 3: 학생 상담 타임라인 UI

**작업내용**

- 학생 선택 시 중앙 영역에 타임라인 뷰 표시 (기록 미선택 상태)
- 세로 타임라인: 날짜 · 상담 제목 · 상담 유형 · 상태 · 미리보기
- 타임라인 항목 클릭 → 해당 기록 상세로 이동
- 기록이 3개 이상이면 "AI 추이 분석" 버튼 표시

### 차수 4: 서버 — 다중 기록 AI 추이 분석 API

**작업내용**

- `POST /api/v1/counseling-records/analyze-trend`
- body: `{ recordIds: string[] }` (같은 학생의 기록 ID 목록)
- 서버에서 각 기록의 transcript를 조합해 하나의 system prompt에 포함
- 프롬프트: "아래는 같은 학생의 여러 차례 상담 원문입니다. 시간 순서로 학생의 변화 추이, 반복되는 이슈, 개선된 점, 주의 필요 사항을 분석해주세요."
- SSE 스트리밍 응답 (기존 chat API 패턴 재활용)
- 토큰 한도 관리: 기록이 많으면 각 기록의 원문을 요약 후 포함

**논의 필요**

- 다중 기록 원문을 전부 보내면 토큰 폭발 가능

**선택지**

- A) 각 기록의 원문 전체 포함 (최대 5개 기록 제한)
- B) 각 기록의 첫 AI 분석 결과만 포함 (이미 요약된 상태)
- C) 서버에서 먼저 각 기록을 요약 → 요약본을 조합해 추이 분석

**추천**: A) 5개 제한 + gpt-5.4-medium의 긴 context 활용. 부족하면 B로 전환.

### 차수 5: 클라이언트 — 추이 분석 스트리밍 UI

**작업내용**

- 타임라인 뷰에서 "AI 추이 분석" 버튼 클릭 → 스트리밍 분석 패널 표시
- 기존 AI 채팅 스트리밍과 동일한 UX (마크다운, 커서, 중지 버튼)
- 분석 완료 후 "클립보드 복사" / "내보내기" 버튼
- 추이 분석도 내보내기 기능과 연동 (77번 백로그)

**논의 필요**: 없음

**추천**: 기존 스트리밍 인프라 재활용.
