# 상담 기록 워크스페이스 방향 정리 BACKLOG

## 차수 1

### 작업내용

- 상담 기록 기능을 `요약기`가 아니라 `고품질 상담 기록 워크스페이스`로 정의한다.
- `AGENTS.md`에 향후 구현 시 따라야 할 제품 우선순위, 품질 기준, 화면 구조 원칙을 반영한다.
- 팀 공유용 기획 방향성 문서를 작성해 문제정의, 사용자 흐름, MVP 범위, 비기능 품질 기준을 고정한다.

### 논의 필요

- 이 기능을 기존 `AI 학생관리 CRM`의 하위 모듈로 둘지, 현재 MVP의 대표 데모 축으로 전면 배치할지 정리가 필요하다.
- 최초 입력을 `인앱 녹음 + 업로드`까지 포함할지, 우선은 `녹음 파일 업로드` 중심으로 갈지 결정이 필요하다.

### 선택지

- `A. 상담 요약 기능 중심`
  녹음 또는 메모를 넣으면 요약과 액션만 생성하는 도구로 범위를 제한한다.
- `B. 상담 기록 워크스페이스 중심`
  녹음본 관리, 고품질 STT, 원문 전체 열람, 구조화 요약, 원문 기반 AI 채팅까지 하나의 작업 화면으로 묶는다.

### 추천

- `B. 상담 기록 워크스페이스 중심`을 추천한다.
- 이유는 신뢰의 source of truth가 `원문 텍스트`에 있고, STT 품질과 원문 열람 가능성이 확보돼야 요약, 검색, AI 채팅, 후속 상담 연결까지 모두 성립하기 때문이다.

## 차수 2

### 작업내용

- 워크스페이스 기준으로 `왼쪽 상담 리스트 / 가운데 원문 중심 본문 / 오른쪽 AI 채팅` IA를 상세화한다.
- 요약 구조를 `핵심 상담 내용 / 학생 문제 포인트 / 보호자 요청사항 / 다음 액션`으로 고정한다.
- `다음 액션은 추천이어야 하며 사람 수정이 가능해야 한다`는 안전 원칙을 명시한다.

### 논의 필요

- 화자 분리, 시간 구간 표시, 근거 문장 연결을 1차 MVP에 포함할지 후속 차수로 미룰지 판단이 필요하다.
- 학생별 누적 히스토리와 검색을 최초 릴리즈에 어디까지 포함할지 정리가 필요하다.

### 선택지

- `A. 원문 열람 + 요약 + 채팅만 우선`
- `B. 여기에 히스토리, 검색, 액션 완료 관리까지 함께 포함`

### 추천

- `A`를 추천하되, 학생별 리스트와 저장 구조는 이후 `B`로 확장될 수 있게 설계한다.
- 이유는 MVP의 본체가 `입력 신뢰성`과 `원문 기반 활용성`이기 때문이다.
