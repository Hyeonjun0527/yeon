# mockdataV2 시뮬레이션 백로그

## 목표

mockdata의 디자인/CSS/브랜딩을 그대로 사용하되, 실제 서비스처럼 상태 전이가 작동하는 시뮬레이션 페이지를 만든다. 서버 없이 목데이터로 동작.

## 1차: 상담 기록 워크스페이스

### 작업내용

- `/mockdata-v2` 라우트 생성
- 로그인 과정 생략, 로그인된 상태로 시작
- 기존 mockdata CSS(`mockdata.module.css`)와 Gnav 컴포넌트 재사용
- 상태 전이가 실제로 작동하는 워크스페이스:
  1. **빈 상태** → 녹음 시작/파일 업로드 CTA
  2. **녹음 중** → 실제 타이머, 정지 버튼
  3. **처리 중** → 프로세싱 애니메이션, 일정 시간 후 자동 완료
  4. **결과** → 전사 텍스트 + AI 요약 + AI 채팅 (목데이터 응답)
  5. **사이드바** → 기록 목록 선택, 새 녹음 버튼

### 논의 필요

- 없음

### 선택지

- A: 한 페이지에 전체 상태 관리 (간단, 빠름)
- B: 페이지별 분리 (구조적이지만 과잉)

### 추천

- A: 한 페이지에 상태 관리. 시뮬레이션이므로 단순하게.

### 사용자 방향

- 쉬운 거부터 차근차근
