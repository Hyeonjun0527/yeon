# 39-social-login-round-2-google-kakao-auth_BACKLOG

## 1차

### 작업내용

- 현재 랜딩 모달의 비활성 버튼 상태를 실제 구글/카카오 OAuth 시작 흐름으로 연결한다.
- 웹 서버에 OAuth 시작, callback 처리, 세션 쿠키 발급, 현재 사용자 조회, 로그아웃 경계를 구현한다.
- DB 스키마를 확장해 소셜 계정 매핑과 세션 저장을 지원하고, 기존 `users`와 충돌 없이 로그인/가입을 처리한다.
- provider mismatch, 이메일 부재, state 검증 실패, 토큰 교환 실패, 프로필 조회 실패, 세션 만료 같은 실패 경계를 실제 서비스 기준으로 정리한다.
- SSR/클라이언트에서 로그인 상태를 일관되게 읽고, 로그인 후 적절한 화면으로 리다이렉트되도록 마무리한다.

### 논의 필요

- 기존 동일 이메일 사용자가 다른 공급자로 처음 들어올 때 자동 링크를 허용할지 여부
- 카카오 계정에서 이메일 미동의 사용자를 어떻게 안내할지 여부

### 선택지

- A. 동일 이메일이면 기존 사용자에 소셜 계정을 자동 링크하고, provider mismatch는 "이미 다른 방식으로 가입됨"으로 차단
- B. 동일 이메일이어도 공급자가 다르면 신규 가입을 막고 명시적 계정 연결 플로우를 따로 만든다

### 추천

- A. 현재 제품 단계에서는 별도 계정 연결 UI가 없으므로 동일 이메일 자동 링크가 가장 작고 안전한 완성 경로다. 다만 이메일이 없거나 검증되지 않은 경우는 링크하지 않고 실패로 처리해야 한다.

### 사용자 방향

- 구글 로그인과 카카오 로그인을 실제 서비스에서 바로 쓸 수 있는 수준으로 완결한다.
