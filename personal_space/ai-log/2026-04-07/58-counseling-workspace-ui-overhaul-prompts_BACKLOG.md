# 상담 기록 워크스페이스 UI 전면 재검토 프롬프트 분해 BACKLOG

## 문서 목적

- 현재 `apps/web/src/features/counseling-record-workspace` 화면을 기능 나열형에서 행동 중심 워크스페이스로 재설계한다.
- 첫 인상은 `여기서 업로드하거나 녹음하면 된다`가 보여야 하고, 기록 선택 전에는 상세/AI 보조 기능을 과하게 펼치지 않는다.
- 이 문서는 기존 `47-counseling-record-workspace-ui_BACKLOG.md`, `57-counseling-workspace-responsive-layout_BACKLOG.md`보다 더 잘게 쪼갠 실행 문서다.
- 각 차수는 AI가 한 번의 프롬프트로 끝까지 처리할 수 있는 정도로만 자른다.

## 공통 목표

- 첫 화면 주인공: `파일 업로드`, `브라우저 녹음`
- 기본 흐름: `업로드/녹음 -> 기록 생성 -> 기록 선택 -> 원문/요약/액션 -> AI 보조`
- 빈 상태는 단순하게 유지하고, 여러 개의 빈 박스를 동시에 보여주지 않는다.
- 긴 설명문, 큰 상태 메트릭, 과한 카드 중첩, 선택 전 AI 패널 상시 노출을 줄인다.
- `왼쪽 생성/탐색`, `가운데 원문`, `오른쪽 AI` 구조는 유지하되, 선택 전에는 밀도를 낮추고 선택 후에만 구조를 완성한다.
