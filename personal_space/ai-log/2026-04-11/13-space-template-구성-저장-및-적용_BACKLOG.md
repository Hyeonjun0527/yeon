# 스페이스 템플릿 구성 저장 및 적용 백로그

## 차수 1

### 작업내용

- 현재 스페이스의 수강생 정보 구성(탭, 표시 여부, 필드 타입/순서)을 템플릿으로 저장하는 서버/클라이언트 경로를 추가한다.
- 저장된 템플릿 목록을 스페이스 설정 drawer 안에서 조회하고, 선택한 템플릿을 현재 스페이스 구성에 한 번에 적용하는 흐름을 구현한다.
- 템플릿 적용 시 기존 커스텀 탭과 커스텀 필드를 정리한 뒤, 시스템 탭 상태와 커스텀 탭/필드를 템플릿 기준으로 재구성한다.

### 논의 필요

- 템플릿을 스페이스 생성 시점에도 바로 선택하게 할지, 현재 차수에서는 스페이스 설정에서 적용하는 범위로 제한할지 결정이 필요하다.
- 템플릿 삭제, 시스템 기본 템플릿 seed, 템플릿 설명 편집은 이번 차수에 포함할지 판단이 필요하다.

### 선택지

1. 스페이스 설정 drawer 내부에서만 템플릿 저장/적용을 먼저 지원한다.
2. 스페이스 생성 플로우와 설정 drawer에 동시에 템플릿 선택 기능을 넣는다.

### 추천

- 선택지 1
- 현재 코드 구조상 settings drawer에 이미 탭/필드 편집 진입점이 있고, 여기서 저장/적용까지 닫으면 변경 범위가 명확하다. 생성 모달까지 넓히면 API와 UX 범위가 커져 첫 landing의 검증 비용이 커진다.
