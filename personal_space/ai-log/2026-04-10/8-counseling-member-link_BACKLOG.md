# 8. 상담 기록 ↔ 수강생 연결 플로우 구축

## 배경

상담 완료 시 `analysis_result` (JSON)가 생성되지만, 이 상담이 어느 스페이스의 어느 수강생 것인지
DB 차원의 연결이 없다. `members.counselingRecordId`가 존재하지만 FK 없는 역방향 1:1 구조로,
한 수강생이 여러 번 상담을 받는 실제 운영 흐름을 담지 못한다.

## 목표

- 상담 기록 → 수강생(member) 연결 (1:N)
- 상담 완료 후 수강생 연결 UI 제공
- 학생 상세에서 해당 수강생의 상담 기록 조회

---

## 1차: 스키마 마이그레이션

### 작업 내용

1. `counseling_records` 테이블에 `member_id` (uuid, nullable) 컬럼 추가
   - FK → `members.id` ON DELETE SET NULL
2. `members` 테이블에서 `counseling_record_id` 컬럼 제거

### 논의 필요

- `member_id` SET NULL vs CASCADE: 수강생 삭제 시 상담 기록은 남겨야 하는가?

### 선택지

- A. SET NULL — 수강생 삭제 시 상담은 미연결 상태로 보존 (기록은 남음)
- B. CASCADE — 수강생 삭제 시 상담도 삭제

### 추천

A. 상담 기록은 운영 이력이므로 수강생 삭제 후에도 보존하는 게 맞음

### 사용자 방향

추천 기준으로 진행 (SET NULL)

---

## 2차: API 계약 및 서버 로직

### 작업 내용

1. `PATCH /api/v1/counseling-records/:recordId` — `memberId` 필드 추가 (연결/해제)
2. `GET /api/v1/spaces/:spaceId/members/:memberId/counseling-records` — 수강생 소속 상담 목록
3. `counseling-records-repository` 쿼리 수정 (member join, filter)
4. `api-contract` 스키마 업데이트

### 논의 필요

- 수강생 연결 시 신규 수강생 생성을 같은 엔드포인트에서 처리할지, 별도 흐름으로 분리할지

### 선택지

- A. 상담 연결 시 `memberId`만 받음 — 수강생 생성은 기존 members API 사용
- B. 상담 연결 API에서 신규 수강생 생성까지 원자적으로 처리

### 추천

A. 단일 책임 원칙 유지. UI에서 순서(생성 → 연결)를 제어

### 사용자 방향

추천 기준으로 진행

---

## 3차: 상담 완료 후 연결 UI

### 작업 내용

상담 완료(분석 완료) 상태에서 "수강생 연결" 액션 제공

**연결 모달 흐름**

```
[수강생 연결] 버튼 클릭
  ├── 기존 수강생 검색 (스페이스 필터 + 이름 검색)
  │     → 선택 → PATCH memberId → 완료
  └── 새 수강생으로 등록
        → 이름 (상담 studentName 기본값)
        → 스페이스 선택 드롭다운 (마지막 사용 스페이스 기본값)
        → POST /members → PATCH memberId → 완료
```

### 논의 필요

- 마지막 선택 스페이스 기본값을 로컬스토리지로 유지할지, 서버에서 최근 스페이스를 내려줄지

### 선택지

- A. 로컬스토리지 — 구현 단순, 기기 간 동기화 없음
- B. 서버 최근 스페이스 — 기기 무관, 구현 복잡

### 추천

A. MVP 단계에서 로컬스토리지로 충분

### 사용자 방향

추천 기준으로 진행

---

## 4차: 학생 상세 → 상담 기록 탭

### 작업 내용

1. 학생 상세 페이지에 "상담 기록" 탭 추가
2. 해당 수강생에 연결된 상담 목록 조회 (날짜, 제목, 분석 요약)
3. 탭에서 기존 미연결 상담을 검색해 연결하는 인라인 액션

### 논의 필요

- 학생 상세가 현재 어디까지 구현되어 있는지 확인 필요

### 선택지

- A. 상담 목록만 조회 (읽기 전용)
- B. 상담 목록 조회 + 미연결 상담 연결 기능

### 추천

B. 운영자가 학생 상세에서 상담을 직접 연결할 수 있어야 실무 활용 가능

### 사용자 방향

추천 기준으로 진행
