# meta-cognitive

## 목표

- 작업이 끝난 뒤 그 경험을 `skills`와 `memory`에 축적해 다음 작업의 품질을 높인다.

## 실행 시점

1. 유의미한 기능 구현 완료 후
2. 버그 수정 완료 후
3. 반복적인 실수가 감지될 때
4. 같은 유형의 리뷰 코멘트가 다시 나왔을 때
5. 사용자가 `/retrospective`를 요청할 때

## 체크포인트

- 잘된 패턴은 재사용 규칙으로 승격할 수 있는가
- 안티패턴이나 버그 패턴으로 기록할 것이 있는가
- 기존 에이전트나 스킬 설명이 현실과 어긋나지 않는가
- 회고 로그에 남길 만한 구현 판단이 있었는가
