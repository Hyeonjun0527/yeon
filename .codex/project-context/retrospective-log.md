# Retrospective Log

## 2026-04-07 소셜 로그인 round 6

- 증상: 구글 로그인 화면을 열어둔 상태에서 카카오 로그인도 시작하거나, 같은 공급자 로그인을 연속으로 두 번 시작하면 첫 번째 콜백이 `invalid_state`로 실패할 수 있었다.
- 실제 원인: OAuth pending 상태를 단일 서명 쿠키 1개로만 관리해, 새 로그인 시작이 이전 `state`를 덮어썼다.
- 더 이른 시점에 잡을 수 있었던 신호: state 검증은 들어가 있었지만, "동시에 두 개 이상의 로그인 시도가 있을 수 있는가"를 플로우 설계 단계에서 따로 점검하지 않았다.
- 다음부터 적용할 규칙: OAuth `state`는 콜백 1회성 값으로만 보지 말고, 같은 브라우저에서 여러 로그인 시도가 겹칠 수 있다는 전제로 다중 pending 상태 또는 공급자별 격리를 먼저 검토한다.
