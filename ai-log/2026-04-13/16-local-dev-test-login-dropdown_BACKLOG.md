## 작업내용

### 1차

- 랜딩 로그인 모달에 로컬 개발환경에서만 보이는 운영자용 테스트 로그인 영역을 추가한다.
- 기존 `/api/auth/dev-login` 라우트는 유지하되, 선택한 `userId`로 로그인할 수 있도록 확장한다.
- 기본 개발자 계정 자동 로그인 동작은 그대로 유지해 Playwright setup과 기존 개발 흐름을 깨지 않는다.
- 로컬 전용 노출 조건과 후보 계정 source of truth를 서버 helper로 묶고, landing page 서버 경계에서 모달로 전달한다.
- 변경 후 관련 테스트와 lint, prettier, typecheck, build를 확인한다.

### 2차

- 로컬 개발 전용 섹션에 `새 계정 생성 후 로그인` 버튼을 추가한다.
- 새 계정은 로컬 테스트용 `dev` identity를 함께 생성해 즉시 로그인 가능 상태로 만든다.
- 기존 선택 로그인과 생성 로그인 흐름을 같은 `/api/auth/dev-login` 라우트에서 query 분기로 처리한다.

## 논의 필요

- 테스트 로그인 후보를 `모든 users`로 볼지, 실제 로그인 이력이 있는 계정만 볼지 기준이 필요하다.

## 선택지

- 선택지 1: `users` 전체를 후보로 보여주고, 선택 로그인 시 필요한 보조 identity를 로컬에서 추가 생성한다.
- 선택지 2: 이미 identity가 연결된 계정만 후보로 노출하고, 나머지는 제외한다.

## 추천

- 선택지 2. 로컬 편의 기능이더라도 기존 사용자/identity 상태를 불필요하게 변형하지 않는 쪽이 안전하고, 현재 세션 검증 규칙과도 충돌이 없다.
