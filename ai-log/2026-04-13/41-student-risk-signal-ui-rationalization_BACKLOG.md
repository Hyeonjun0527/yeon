# 수강생 위험도/AI 위험 신호 UI 정리 BACKLOG

## 결정 요약

- `운영 상태` 섹션의 `위험도` row는 제거한다.
- `상담 기록` 섹션의 `AI 위험 신호` row만 유지한다.
- `AI 위험 신호` row 오른쪽에 `?` 버튼을 두고 hover 시 고정 설명을 노출한다.
- 설명 문구는 `상담을 분석하여 위험신호를 결정합니다.` 계열의 짧은 안내로 둔다.
- 목록 카드에서는 `aiRiskSummary`를 제거하고, 대신 상담 기록 기반 `AI 위험 신호`를 보여준다.
- 목록 카드의 `위험도 배지`는 기존 `aiRiskLevel`을 그대로 유지한다.
- `aiRiskSummary`는 상세/목록 모두에서 제거한다.

## 1차

### 작업내용

- `tab-member-overview.tsx`에서 `운영 상태 > 위험도` row 제거
- `상담 기록 > AI 위험 신호` row에 툴팁 트리거 추가
- `AI 위험 신호` empty/filled 상태 문구와 행 레이아웃 정리
- `aiRiskSummary` note 의존 제거

### 논의 필요

- 툴팁 UI를 기존 공용 컴포넌트로 재사용할지, 이 파일 내부의 경량 hover 패턴으로 둘지

### 선택지

- 공용 툴팁 사용
- 로컬 hover 패턴 사용

### 추천

- 로컬 hover 패턴 사용
- 현재 범위는 단일 화면 조정이고, 기능 요구가 단순해서 공용 추상화보다 안전하다.

### 사용자 방향

## 2차

### 작업내용

- `student-list-screen.tsx` 카드 보기에서 `aiRiskSummary` 제거
- 카드 하단 보조 문구를 `AI 위험 신호` 표시로 교체
- dense/card 뷰의 정보 위계를 다시 맞춰 카드 높이 증가를 최소화
- `aiRiskLevel` 배지는 현행 유지

### 논의 필요

- 카드에서 `AI 위험 신호`를 텍스트 1줄로 자를지, 칩 형태 1~2개만 노출할지

### 선택지

- 쉼표 텍스트 1줄
- 칩 1~2개 + 나머지 `+N`

### 추천

- 쉼표 텍스트 1줄
- 현재 카드 밀도를 유지하면서도 구현 리스크가 가장 작다.

### 사용자 방향

## 3차

### 작업내용

- `aiRiskSummary` 제거가 실제로 상세/목록 외 다른 화면에 영향을 주는지 점검
- 관련 타입과 테스트 정리
- 최소 회귀 검증
  - 상세 개요
  - 목록 카드 보기
  - dense 보기

### 논의 필요

- 타입에서 `aiRiskSummary`를 완전히 제거할지, 서버 계약은 유지하고 UI만 미사용으로 둘지

### 선택지

- UI만 미사용
- 타입/서버 계약까지 제거

### 추천

- 이번 landing은 UI만 미사용
- 서버 계약 삭제는 범위를 키우고 회귀 리스크가 커진다.
