# 수강생 추가 legacy 위험도 입력 제거

## 작업내용

- 수강생 추가 화면에서 legacy 위험도 입력 UI를 제거한다.
- 신규 수강생 생성 API payload에서 `initialRiskLevel`을 보내지 않도록 정리한다.

## 논의 필요

- 위험도 legacy 필드를 다른 화면과 계약에서 언제 완전히 제거할지는 별도 작업으로 분리한다.

## 선택지

- A. 추가 화면만 숨기고 payload는 유지한다.
- B. 추가 화면과 생성 payload 둘 다 제거한다.

## 추천

- B. 사용자에게 보이지 않는 값을 생성 시점에 계속 보내는 이유가 없고, 현재 제품 개념과도 맞지 않는다.
