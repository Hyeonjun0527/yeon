# AI 부트캠프 학생 관리 플랫폼 기획서

## 프로젝트 한 줄 정의

> **"기존 툴을 그대로 쓰면서, 흩어진 데이터를 연동해 부트캠프 학생 관리의 품질을 완전히 바꾼다"**

---

## 배경 및 문제 인식

국내 개발 부트캠프는 짧은 기간(3~6개월) 안에 수십 명의 수강생을 관리해야 하는 고강도 환경이다. 강사는 수업, 과제 피드백, 운영 행정을 동시에 처리하고, 수강생 역시 매일 쏟아지는 강의와 과제를 소화하느라 여유가 없다.

이 환경에서 학생 관리는 대부분 이렇게 이루어진다:

- 출결 → 구글 폼
- 과제 제출 → 구글 폼 or 엑셀
- 수강생 정보 → 엑셀
- 수업 자료 → 노션
- 소통 → 슬랙
- 코드 제출 → 깃허브

데이터는 존재하지만, 흩어져 있다. 강사가 특정 수강생의 상태를 파악하려면 여러 툴을 직접 뒤져야 한다. 결국 학생 관리는 감과 기억에 의존하게 된다.

---

## 핵심 가치

### 메인: 학생 관리 리포트 품질 업그레이드

기존에는 불가능했던 것들이 가능해진다:

- 수강생 한 명의 출결, 과제 제출 패턴, 자료 열람, 코드 활동이 **한 곳에서** 보인다
- 강사가 수십 개의 툴을 뒤지지 않아도, 수강생의 이번 주 상태를 한눈에 파악한다
- 입학 상담에서 파악한 수강생의 배경과 목표가 수강 중 데이터와 연결되어 맥락 있는 관리가 가능해진다
- 수강생 본인도 자신의 진도와 상태를 객관적으로 확인할 수 있다

### 덤: 위험 수강생 조기 감지

학생 관리 품질이 올라가면 자연스럽게 따라오는 효과다:

- 행동 로그 패턴에서 위험 신호를 AI가 먼저 감지
- 강사에게 선제 알림 → 수강생이 말하기 전에 먼저 다가갈 수 있음
- 중도 이탈률 감소로 이어짐

## 사용자

**주 사용자: 강사 / 부트캠프 운영자**

- 수강생 상태를 빠르게 파악하고 싶다
- 추가 행동 없이 데이터가 자동으로 모이길 원한다
- 위험 수강생을 미리 알고 싶다

**부 사용자: 수강생**

- 내가 잘 하고 있는지 불안하다
- 내 진도와 상태를 객관적으로 보고 싶다

## 핵심 기능

### 1. 기존 툴 연동 (데이터 자동 수집)

강사도, 수강생도 아무것도 추가로 하지 않아도 된다. 기존에 쓰던 툴에 yeon을 연결하기만 하면 데이터가 자동으로 수집된다.

**1차 연동 (MVP)**

- 구글 폼 / 구글 시트 → 출결, 과제 제출 타임스탬프

**확장 연동**

- 노션 → 수업 자료 열람 여부
- 슬랙 → 질문 빈도, 반응 패턴
- 깃허브 → 커밋 빈도, 마지막 푸시 시각
- 원드라이브 → 파일 제출 기록

**선택 이유**: 한국 부트캠프 현장에서 구글 폼/시트가 가장 보편적으로 사용된다. 새로운 툴 도입 없이 기존 방식 그대로 유지하면서 연동만 하면 되기 때문에 도입 장벽이 최소화된다.

### 2. 입학 상담 녹음 AI 분석

입학 상담은 수강생을 가장 처음 파악하는 순간이다. 이때의 정보가 수강 내내 맥락으로 활용되어야 한다.

- 상담 녹음 업로드 → AI 자동 분석
- 개발 경험 수준, 목표 명확성, 불안 수준, 학습 동기 파악
- 수강생 초기 프로파일 자동 생성
- 수강 중 데이터와 연결되어 맥락 있는 관리 가능

### 3. 통합 학생 리포트

흩어진 데이터를 한 곳에서 확인한다.

**강사용 리포트**

- 전체 수강생 한눈에 보기
- 수강생별 출결, 과제 완료율, 활동 패턴
- 입학 상담 프로파일 + 현재 상태 비교
- 위험 신호 수강생 하이라이트

**수강생용 리포트**

- 내 출결, 과제 완료율
- 이번 주 강사 피드백
- 입학 목표 대비 현재 진행 상황

### 4. RAG 기반 위험 신호 감지

- 행동 로그 패턴 + 입학 프로파일 결합 분석
- 과거 수강생 데이터 RAG 검색으로 유사 패턴 감지
- "이런 패턴의 수강생은 4주차에 이탈한 사례 많음" 근거 기반 알림
- 수강생이 많을수록, 운영 기간이 길수록 정밀도 향상

## 기술 구조

```
[기존 툴]                    [yeon]
구글 시트 ──────────────▶
노션        연동 커넥터    데이터 수집
슬랙 ──────────────────▶  + 정규화
깃허브                        │
                              ▼
입학 상담 녹음 ────────▶  AI 분석 엔진
                         (LLM + RAG)
                              │
                              ▼
                    통합 학생 리포트
                    위험 신호 알림
```

## 차별점

| 기존 방식                    | yeon                          |
| ---------------------------- | ----------------------------- |
| 데이터가 여러 툴에 흩어짐    | 한 곳에서 통합 확인           |
| 강사의 감과 기억에 의존      | 데이터 기반 객관적 관리       |
| 위험 신호를 늦게 알아챔      | AI가 먼저 감지                |
| 새 툴 도입 시 업무 방식 변경 | 기존 툴 그대로 연동           |
| 수강생 수 무관하게 동일      | 데이터 누적될수록 정밀도 향상 |

## 구현 순서 (백로그)

- [ ] 구글 시트 연동 커넥터
- [ ] 출결 / 과제 제출 로그 자동 수집
- [ ] 강사용 수강생 리포트 (기본)

### 2차수

- [ ] 수강생용 리포트
- [ ] RAG 기반 위험 신호 감지 + 강사 알림
- [ ] 노션 / 슬랙 연동

### 3차수

- [ ] 깃허브 / 원드라이브 연동
- [ ] 수강생 축적 데이터 기반 예측 정밀도 향상
- [ ] 부트캠프간 벤치마킹 데이터 제공

## 공모전 제출용 요약

**프로젝트명**: (미정)

**한 줄 설명**
기존 툴을 바꾸지 않고, 흩어진 부트캠프 데이터를 연동해 학생 관리 품질을 완전히 업그레이드하는 AI 플랫폼

**사용자**: 부트캠프 강사 / 운영자

**핵심 문제**
부트캠프의 학생 관리 데이터는 구글 시트, 노션, 슬랙, 깃허브에 흩어져 있다. 강사는 수강생 상태를 파악하기 위해 여러 툴을 직접 뒤져야 하고, 결국 감과 기억에 의존하게 된다. 위험 수강생은 말하지 않고, 강사는 너무 늦게 알아챈다.

**솔루션**
아무도 추가 행동을 하지 않아도 된다. 기존 구글 시트에 yeon을 연결하면 출결, 과제, 활동 로그가 자동 수집된다. 입학 상담 녹음은 AI가 분석해 수강생 프로파일을 만든다. 이 두 데이터가 합쳐져 통합 학생 리포트가 생성되고, RAG 기반으로 위험 신호를 조기 감지한다. 수강생이 쌓일수록 더 정밀해진다.
