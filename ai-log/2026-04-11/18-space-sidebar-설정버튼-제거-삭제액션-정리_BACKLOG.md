## 1차: 사이드바 액션 역할 정리

### 작업내용

- 스페이스 리스트의 선택 행에서 inline 설정 버튼을 제거한다.
- 스페이스 설정은 상단의 "스페이스 설정" 진입점으로 단일화한다.
- 선택 행에는 삭제 액션만 남겨 역할을 분리한다.

### 논의 필요

- 삭제 버튼 노출을 선택된 스페이스에만 둘지 여부

### 선택지

A. 선택된 스페이스에만 삭제 버튼 노출  
B. 모든 스페이스 행에 항상 삭제 버튼 노출

### 추천

A — 현재 UI 밀도를 유지하면서 실수 클릭 위험을 줄일 수 있다.

## 2차: 스페이스 삭제 경로 보강

### 작업내용

- 현재 없는 스페이스 삭제 API를 추가한다.
- 삭제 후 스페이스 목록, 선택 상태, 상세 라우트가 깨지지 않도록 provider와 레이아웃 상태를 함께 정리한다.

### 논의 필요

- 삭제 확인 문구와 삭제 후 fallback 선택 규칙

### 선택지

A. 삭제 후 전체 수강생(null)로 이동  
B. 삭제 후 남은 첫 스페이스로 자동 이동

### 추천

A — 예측 가능하고 부수효과가 가장 적다.
