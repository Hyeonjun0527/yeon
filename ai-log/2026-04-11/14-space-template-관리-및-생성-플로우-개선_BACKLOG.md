# 스페이스 템플릿 관리 및 생성 플로우 개선 백로그

## 차수 1

### 작업내용

- 스페이스 설정 drawer 안에서 템플릿을 저장/적용하는 수준을 넘어, 사용자 정의 템플릿의 설명과 삭제 같은 기본 관리 액션을 제공한다.
- 새 스페이스 생성 모달의 템플릿 선택 UX를 개선해 템플릿별 구조 차이를 더 쉽게 이해하고 고르게 한다.
- 템플릿 목록 API 응답을 UI 친화적인 summary 형태로 정리하고, space settings와 create-space modal이 같은 정보를 공유하도록 맞춘다.

### 논의 필요

- 사용자 정의 템플릿 수정 범위를 설명 편집까지만 열지, 이름 변경까지 함께 허용할지 결정이 필요하다.
- 템플릿 삭제를 soft delete로 둘지, 현재처럼 hard delete로 유지할지 정책 정리가 필요하다.

### 선택지

1. 설명 편집 + 삭제 + 요약 정보 노출까지 우선 지원한다.
2. 템플릿 생성/적용 외 모든 관리 기능을 추후 별도 템플릿 관리 화면으로 미룬다.

### 추천

- 선택지 1
- 템플릿을 실제 운영에서 쓰려면 저장만 가능한 상태보다 최소한의 관리 기능이 바로 필요하다. 별도 관리 화면 없이 drawer와 생성 모달 안에서 닫으면 지금 구조와도 잘 맞는다.
