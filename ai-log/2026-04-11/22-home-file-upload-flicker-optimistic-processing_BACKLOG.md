## 1차: 업로드 진입 직후 화면 상태 안정화

### 작업내용

- 파일 업로드 흐름에서도 녹음 흐름과 동일하게 임시 processing 레코드를 즉시 생성한다.
- 서버 응답 전까지 빈 화면이나 멤버 패널 해제가 끼어들지 않도록 view 전환을 안정화한다.
- 실제 서버 레코드가 도착하면 임시 레코드를 교체하고, 실패 시 같은 임시 레코드에 오류 상태를 반영한다.

### 논의 필요

- 없음

### 선택지

A. 서버 응답 뒤에만 레코드 추가  
B. 파일 선택 즉시 임시 processing 레코드 추가

### 추천

B — 현재 flicker 원인이 응답 대기 구간의 화면 공백이므로, 처리중 상태를 즉시 올리는 편이 가장 직접적이다.
