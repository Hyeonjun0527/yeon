# Student report workspace V1 백로그

## 차수 1

### 작업내용

- 학생 상세 `리포트` 탭을 단순 통계 화면에서 문서 생성 워크스페이스로 전환한다.
- member 기준 상담기록 목록 API를 리포트 소스로 사용하고, 리포트 종류 선택 / 포함 섹션 설정 / 미리보기 / 복사·다운로드 동작을 한 화면에서 제공한다.
- 기존 `activity-logs` 의존은 V1에서 제거하거나 보조 정보로 격하시켜, route 부재 때문에 report 탭이 비는 문제를 없앤다.
- 이후 상담기록 화면과 공유할 수 있도록 리포트 데이터 조합/문서 문자열 생성 로직을 별도 유틸로 분리한다.

### 논의 필요

- V1에서 DOCX까지 바로 지원할지, 우선 Markdown/TXT export와 preview를 먼저 안정화할지 결정이 필요하다.
- 리포트 유형별 템플릿을 space-level 설정으로 끌어올릴지, 우선 탭 내부 로컬 설정으로 둘지 후속 판단이 필요하다.

### 선택지

1. 현재 TabReport 통계 카드만 조금 보강한다.
2. 학생 상세 리포트 탭을 문서 생성 워크스페이스로 전환하고, 상담기록 기반 preview/export를 먼저 완성한다.

### 추천

- 선택지 2
- 사용자가 말한 리포트 가치(문서 생성, 설정, 프리뷰, 원하는 결과물 선택)는 단순 통계 카드와 맞지 않는다. V1에서 학생 상세 탭을 실제 report workspace로 바꿔야 이후 상담기록 진입점과 공통 엔진으로 확장하기 쉽다.
