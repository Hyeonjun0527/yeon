# External import entry 단일화 백로그

## 차수 1

### 작업내용

- 가져오기 모달에서 중복된 `OneDrive에서 가져오기` / `Google Drive에서 가져오기` 진입 버튼을 하나의 `외부에서 가져오기` CTA로 단일화한다.
- 실제 공급자 선택은 이미 존재하는 downstream cloud import 흐름 안에서 유지하고, 사용자는 진입 후에 Google Drive / OneDrive 중 하나를 고를 수 있게 정리한다.
- 현재 로컬 업로드 흐름과 draft 복구 흐름은 건드리지 않고, entry 레이어의 시각적 중복만 제거한다.

### 논의 필요

- 외부 import 진입 직후 공급자 선택을 탭으로 둘지, segmented control처럼 더 가볍게 보일지 후속 미세조정 여지가 있다.
- provider 연결 전 상태와 연결 후 파일 탐색 상태를 같은 헤더 구조로 보여줄지 검토가 필요하다.

### 선택지

1. 현재처럼 provider별 진입 CTA 두 개를 유지한다.
2. entry CTA는 하나로 줄이고 provider 선택은 다음 단계에서 유지한다.

### 추천

- 선택지 2
- 지금 구조는 동일한 작업을 시작하는 CTA가 둘이라 시각적 우선순위가 흐려진다. entry는 하나로 단순화하고, 실제 provider 차이는 다음 단계에서 선택하게 두는 편이 정보 구조와 행동 흐름이 더 자연스럽다.
