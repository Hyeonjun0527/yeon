# mobile landing page redesign backlog

## 작업내용

### 1차

- 현재 랜딩 페이지의 모바일 레이아웃 붕괴 원인을 정리한다.
- hero, stats, features, flow, CTA 섹션을 모바일 우선 레이아웃으로 재설계한다.

### 2차

- 모바일에서 과도한 데스크톱 조형 요소가 겹치지 않도록 hero 배경과 텍스트 계층을 다시 맞춘다.
- 제목, 보조문구, CTA를 한 손 화면에서 읽기 쉬운 리듬으로 재배치한다.

### 3차

- stats와 feature 카드를 모바일에서 1열 또는 2열 균형 구조로 재정렬한다.
- 텍스트 줄바꿈, 카드 높이, 버튼 폭, 섹션 간격을 손본다.

### 4차

- Playwright 모바일 viewport로 실제 화면을 검증하고, 타입체크 및 diff 상태를 점검한다.

## 논의 필요

- 모바일 hero에서 배경 조형을 얼마나 강하게 남길지.
- CTA를 1개 primary 중심으로 단순화할지, 데모/로그인 2개를 유지할지.

## 선택지

1. 기존 데스크톱 레이아웃 축소판
   - 지금 구조를 유지하면서 간격만 줄이는 방식
2. 모바일 전용 재구성
   - hero hierarchy, 카드 배치, CTA 구조를 모바일 기준으로 다시 짜는 방식

## 추천

- 이번 차수는 **모바일 전용 재구성**으로 진행한다.
- 데스크톱 조형을 그대로 축소하지 않고, 읽기 흐름과 행동 유도 중심으로 다시 설계한다.

## 사용자 방향

- 모바일 랜딩페이지를 읽기 쉽고 정렬이 맞는 형태로 전면 재디자인 진행
