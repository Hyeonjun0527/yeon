# 14-student-management-layout-sidebar-orchestration-refactor BACKLOG

## 차수 1

### 작업내용

- `app/home/student-management/layout.tsx`에 뭉쳐 있는 스페이스 선택, shift 다중 선택, 컨텍스트 메뉴, 이름 변경/삭제 액션, 생성 모달 진입 상태를 분리한다.
- 현재 사이드바 UI와 라우팅, 삭제/이름변경 동작, draft CTA 동작은 그대로 유지한다.
- 레이아웃 파일은 route shell과 조합에 집중하고, 상태 계산/액션 실행은 별도 훅과 유틸로 옮긴다.

### 논의 필요

- 이 차수에서 rename/delete 모달 자체까지 분리할지, 우선 상태/액션만 분리할지 판단이 필요하다.
- space selection 로직을 이후 상담 홈 사이드바와 공용화할지 여부는 후속에서 검토할지 정해야 한다.

### 선택지

1. `layout.tsx` 내부에서 함수명만 정리하고 구조는 유지한다.
2. space selection/action 상태와 API 액션을 전용 훅/유틸로 빼고 레이아웃은 조합만 담당하게 만든다.
3. 사이드바 UI까지 별도 컴포넌트 트리로 즉시 분해한다.

### 추천

- 2번으로 간다. 현재 리스크는 레이아웃 파일이 route shell 역할을 넘어서 선택 상태, 삭제/이름변경 액션, 가져오기 진입 흐름까지 한꺼번에 들고 있다는 점이다. 먼저 행동 보존형 분리로 상태와 액션 책임을 줄인 뒤, UI 컴포넌트 분해는 다음 차수에서 판단하는 것이 안전하다.
