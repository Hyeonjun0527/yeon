# 8-student-management-multi-draft-recovery BACKLOG

## 차수 1

### 작업내용

- 학생관리 사이드바의 단일 임시초안 CTA를 다건 draft 전제에 맞는 가져오기 작업 진입점으로 교체한다.
- 스페이스 만들기 > AI 가져오기 모달 안에서 local import draft 목록을 보여주고, 사용자가 원하는 draft를 직접 선택해 복구하게 만든다.
- draft 삭제/가져오기 완료 후 목록과 재진입 상태가 즉시 갱신되도록 정리한다.

### 논의 필요

- draft 목록을 import 모달 상단의 보조 섹션으로 둘지, 별도 탭/패널로 분리할지 UX 기준이 필요하다.
- 사이드바에서는 개수만 노출할지, 최근 항목 일부를 미리 보여줄지 결정이 필요하다.

### 선택지

1. 사이드바에 최근 1개만 보여주고 나머지는 포기한다.
2. 사이드바에는 가져오기 작업 개수만 보여주고, import 모달 안에서 목록을 선택한다.
3. 별도 draft 관리 페이지를 새로 만든다.

### 추천

- 2번으로 간다. 사이드바는 좁아서 다건 draft 세부 정보를 담기 부적합하고, 실제 복구 행동은 import 문맥 안에서 일어나야 한다. 개수 기반 진입점 + 모달 내부 목록이 현재 구조에서 가장 자연스럽다.
