# 53. 상담 원본 음성 스트리밍 응답

## 작업내용

### 1차

- R2 원본 음성 응답을 바이트 버퍼 대신 스트리밍으로 전달한다.
- Range 요청은 유지하면서 브라우저 오디오 플레이어가 긴 파일을 메모리 폭증 없이 재생하도록 개선한다.
- 기존 전사용 다운로드는 그대로 유지한다.

## 논의 필요

- 현재는 프라이빗 서버 프록시를 유지한다.
- presigned URL 직행은 보안 정책과 접근 로그 기준이 정리된 뒤 후순위로 본다.

## 선택지

- 선택지 A: 앱 서버가 R2 body를 스트리밍 프록시한다.
- 선택지 B: presigned URL로 브라우저가 직접 가져오게 한다.

## 추천

- 선택지 A.
- 현재 인증 경계와 접근 제어를 유지하면서도 긴 오디오 메모리 부담을 크게 줄일 수 있다.
