# 40-social-login-round-3-service-hardening_BACKLOG

## 1차

### 작업내용

- 실제 배포와 로컬 실행에서 구글/카카오 OAuth 환경변수가 빠지지 않도록 compose, 앱 예시 env, 배포 문서를 보완한다.
- 이미 연결된 소셜 계정이 재로그인할 때 검증되지 않은 이메일로 기존 사용자 이메일이 덮어써지지 않도록 로그인 병합 규칙을 강화한다.
- 세션 조회 API에서 무효하거나 만료된 세션 쿠키를 정리해 인증 상태가 false일 때 브라우저 쿠키가 계속 남는 문제를 줄인다.

### 논의 필요

- 세션을 조회할 때 만료 시간을 슬라이딩 갱신할지 여부

### 선택지

- A. 이번 차수에서는 stale 쿠키 정리와 최근 접근 시각 갱신만 넣고, 세션 TTL 자체는 고정 유지
- B. 이번 차수에서 세션 TTL도 매 요청마다 연장

### 추천

- A. 현재 서비스에는 세션 정책 변경보다 일관된 정리 동작이 먼저다. TTL을 고정한 채 stale 정리와 접근 기록만 넣는 편이 작고 안전하다.
