# 51. 상담 기록 원본 음성 R2 저장소 전환

## 작업내용

### 1차

- 상담 원본 음성 저장소를 로컬 디스크에서 Cloudflare R2로 전환한다.
- 업로드 시 R2에 안전하게 저장하고, 기존 DB의 `audioStoragePath`는 R2 object key를 source of truth로 사용한다.
- 원본 음성 다운로드 route가 R2에서 파일을 읽도록 변경한다.
- STT 전사 시 R2 원본을 임시 로컬 파일로 받아 기존 chunking/ffmpeg/OpenAI 전사 흐름을 유지한다.
- 환경변수 예시와 로컬 설정 가이드를 R2 기준으로 갱신한다.

### 2차

- 업로드/다운로드 실패 메시지를 R2 기준으로 더 명확하게 다듬는다.
- 필요 시 presigned URL 또는 multipart upload로 확장 가능한 구조인지 점검한다.

## 논의 필요

- 로컬 개발에서도 R2를 필수로 강제할지, 로컬 fallback을 남길지 결정이 필요하다.
- 현재는 서버 프록시로만 원본 음성을 제공할지, 이후 presigned URL로 전환할지 결정이 필요하다.

## 선택지

- 선택지 A: R2를 필수 저장소로 강제하고 env 누락 시 업로드를 막는다.
- 선택지 B: R2 우선, env 없으면 로컬 디스크 fallback을 유지한다.

## 추천

- 선택지 A.
- 지금 사용자 방향이 `실제 운영 저장소를 R2로 붙인다`는 쪽으로 명확하고, fallback을 남기면 운영 결함을 늦게 발견할 가능성이 크다.
