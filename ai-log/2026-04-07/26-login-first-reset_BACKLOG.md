# 26-login-first-reset_BACKLOG

## 작업내용

- 현재 루트, `contest`, `student-management`에 걸려 있는 데모성 프론트 화면을 전부 제거한다.
- 홈은 로그인 우선 방향만 남기는 최소 진입 화면으로 교체하고, 기존 학생 큐·학생 상세·오늘 액션보드·최근 기록 중심 UI는 모두 걷어낸다.
- `packages/design-tokens`와 전역 토큰성 CSS 변수를 제거해 디자인 토큰 의존을 없앤다.

## 논의 필요

- 로그인 화면을 실제 인증까지 이번 차수에 붙일지, 우선 진입용 정적 화면으로 리셋할지

## 선택지

- A. 로그인 우선 정적 진입 화면으로 먼저 리셋하고 실제 인증은 다음 차수로 분리
- B. 삭제와 동시에 실제 로그인 세션까지 한 번에 구현

## 추천

- A. 현재 요청의 핵심은 잘못된 제품 흐름으로 쌓인 프론트 화면을 즉시 철거하는 것이다. 우선 잘못된 화면을 제거하고, 로그인 > 프로젝트 생성 > 학생 파일 업로드 흐름은 다음 차수에서 안전하게 다시 쌓는 편이 범위와 위험이 맞다.

## 사용자 방향

- 현재 작업한 프론트 화면과 디자인 토큰을 모두 지우고 로그인부터 다시 시작하는 구조로 초기화
