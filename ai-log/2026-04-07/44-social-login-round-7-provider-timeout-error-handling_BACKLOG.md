# 44-social-login-round-7-provider-timeout-error-handling_BACKLOG

## 1차

### 작업내용

- 구글/카카오 토큰 교환과 프로필 조회 요청에 타임아웃과 네트워크 오류 매핑을 추가한다.
- 공급자 호출이 끊기거나 오래 멈춰도 `server_error`로 뭉개지지 않고, 실제 실패 단계에 맞는 인증 오류로 처리되게 만든다.

### 논의 필요

- 공급자 요청 타임아웃을 짧게 둘지, 다소 여유 있게 둘지 여부

### 선택지

- A. 10초 내외의 명시적 타임아웃을 두고 실패 단계를 보존한다
- B. fetch 기본 동작에 맡기고 응답이 돌아올 때까지 기다린다

### 추천

- A. 외부 OAuth 공급자 장애나 네트워크 흔들림에서 요청이 무기한 매달리지 않게 하면서, 운영 로그와 사용자 오류 메시지도 더 정확해진다.
