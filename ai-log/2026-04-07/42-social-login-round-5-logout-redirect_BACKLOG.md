# 42-social-login-round-5-logout-redirect_BACKLOG

## 1차

### 작업내용

- 로그아웃 폼의 POST 요청 이후 홈으로 이동할 때 브라우저가 POST를 재전송하지 않도록 리다이렉트 상태 코드를 바로잡는다.
- 소셜 로그인으로 발급된 세션을 삭제한 뒤, 일관되게 GET 기준 랜딩 페이지로 복귀하도록 인증 종료 흐름을 마무리한다.

### 논의 필요

- 로그아웃 완료 후 항상 홈으로 보낼지, 직전 페이지 또는 별도 완료 화면으로 보낼지 여부

### 선택지

- A. 현재처럼 홈으로 보내되 POST 이후에는 303으로 GET 전환을 강제한다
- B. 응답 본문만 반환하고 프론트가 직접 이동을 결정한다

### 추천

- A. 기존 사용자 경험을 유지하면서도 잘못된 POST 재전송 가능성만 제거하는 가장 작은 안전 수정이다.
