# 50 상담 기록 화자 분리 품질 보강 BACKLOG

## 차수 1

### 작업내용

- 상담 전사 기본 모델을 화자 분리 가능한 모델 기준으로 조정한다.
- 모델 미지원 또는 권한 문제 시 일반 전사 모델로 안전하게 fallback 한다.
- 저장되는 `sttModel`이 실제 사용 모델을 반영하도록 수정한다.
- env 예시와 화면 카피를 현재 동작에 맞게 정리한다.

### 논의 필요

- 기본값을 diarization 모델로 두는 것이 비용/속도 측면에서 팀 운영 기준과 맞는지

### 선택지

- 선택지 A: 기본을 `gpt-4o-transcribe-diarize`, fallback 을 `gpt-4o-transcribe`로 둔다.
- 선택지 B: 기본을 계속 `gpt-4o-transcribe`로 두고 운영 환경에서만 수동 전환한다.

### 추천

- 선택지 A
- 상담 기록 워크스페이스에서 화자 구분은 실사용 품질에 직접 연결되고, fallback 을 두면 안전성도 유지된다.
